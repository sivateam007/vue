Vue HTTP கோரிக்கைகள்

HTTP கோரிக்கை என்பது ஒரு வாடிக்கையாளர் மற்றும் சேவையகத்திற்கு இடையேயான தகவல் தொடர்பின் ஒரு பகுதியாகும்.

வாடிக்கையாளர் சேவையகத்திற்கு ஒரு HTTP கோரிக்கையை அனுப்புகிறார், இது கோரிக்கையைக் கையாளுகிறது மற்றும் ஒரு HTTP பதிலைத் தருகிறது.

HTTP - ஹைப்பர் டெக்ஸ்ட் டிரான்ஸ்பர் ப்ரோட்டோகால்

HTTP என்பது ஹைப்பர் டெக்ஸ்ட் டிரான்ஸ்பர் ப்ரோட்டோகால் என்று பொருள்.

நாம் இணையத்தில் உலாவும் போது நமது உலாவி பின்னணியில் எப்போதும் HTTP கோரிக்கைகளைச் செய்கிறது. நாம் ஒரு இணையப் பக்கத்தை அணுகும் போது, நமது உலாவி (வாடிக்கையாளர்) சேவையகத்திற்கு பல HTTP கோரிக்கைகளை அனுப்புகிறது, எங்களுக்குத் தேவையான பக்கத்தை அனைத்து தொடர்புடைய கோப்புகள் மற்றும் தரவுகளுடன் HTTP பதில்களாக அனுப்புகிறது.

🌐 HTTP பற்றி:

HTTP என்பது வலைத் தரவுகளை பரிமாறிக்கொள்ளப் பயன்படும் ப்ரோட்டோகால் ஆகும். இது வாடிக்கையாளர்-சேவையக மாதிரியில் செயல்படுகிறது, அங்கு ஒரு வலை உலாவி வாடிக்கையாளராக செயல்படுகிறது மற்றும் வலைச் சேவையகம் சேவையகமாக செயல்படுகிறது.

'fetch' முறை

Vue இல் ஒரு சேவையகத்திலிருந்து தரவைப் பெற JavaScript fetch() முறையைப் பயன்படுத்தலாம்.

இந்த டுடோரியலில் fetch() முறையைப் பயன்படுத்தும்போது HTTP கோரிக்கை முறையைக் குறிப்பிட மாட்டோம், அதாவது பின்னணியில் பயன்படுத்தப்படும் இயல்புநிலை கோரிக்கை முறை GET ஆகும்.

fetch() முறை தரவை எங்கிருந்து பெறுவது என்பதை அறிய ஒரு URL முகவரியை ஒரு வாதமாக எதிர்பார்க்கிறது.

HTTP GET கோரிக்கையை அனுப்புவதற்கும், HTTP பதிலாக தரவைப் பெறுவதற்கும் fetch() முறையைப் பயன்படுத்தும் ஒரு எளிய எடுத்துக்காட்டு இங்கே உள்ளது. இந்த வழக்கில் கோரிக்கை செய்யப்பட்ட தரவு உள்ளூர் கோப்பு file.txt இல் உள்ள உரையாகும்:

📝 எடுத்துக்காட்டு

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

App.vue:

<template>
  <div>
    <button @click="fetchData">Fetch Data</button>
    <p v-if="data">{{ data }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      data: null,
    };
  },
  methods: {
    fetchData() {
      const response = fetch("file.txt");
      this.data = response;
    }
  }
};
</script>

மேலே உள்ள எடுத்துக்காட்டில், "[object Promise]" என்ற முடிவை மட்டுமே பெறுகிறோம், ஆனால் அது நமக்கு வேண்டியதல்ல.

fetch() ஒரு வாக்குறுதி அடிப்படையிலான முறையாக இருப்பதால் இந்த முடிவைப் பெறுகிறோம், இது ஒரு வாக்குறுதிப் பொருளை வழங்குகிறது. எனவே fetch() முறையின் முதல் வருமானம் HTTP கோரிக்கை அனுப்பப்பட்டுள்ளது என்று அர்த்தமுள்ள ஒரு பொருள் மட்டுமே. இது "நிலுவையில்" உள்ள நிலையாகும்.

fetch() முறை உண்மையில் நமக்குத் தேவையான தரவைப் பெறும்போது, வாக்குறுதி நிறைவேற்றப்படுகிறது.

async மற்றும் await பயன்படுத்துதல்

நமக்குத் தேவையான தரவுடன் பதில் நிறைவேறுவதற்குக் காத்திருக்க, fetch() முறைக்கு முன்னால் await ஆபரேட்டரைப் பயன்படுத்த வேண்டும்:

const response = await fetch("file.txt");

ஒரு முறைக்குள் await ஆபரேட்டர் பயன்படுத்தப்படும்போது, async ஆபரேட்டருடன் முறை அறிவிக்கப்பட வேண்டும்:

async fetchData() {
  const response = await fetch("file.txt");
  this.data = response;
}

async ஆபரேட்டர் உலாவிக்கு முறை அசிங்க்ரோனஸ் என்று கூறுகிறது, அதாவது அது ஏதாவது ஒன்றிற்காக காத்திருக்கிறது, மற்றும் முறை முடிவடையும் வரை காத்திருக்கும் போது உலாவி மற்ற பணிகளைச் செய்யத் தொடரலாம்.

இப்போது நாம் பெறுவது ஒரு "பதில்" மட்டுமே, "வாக்குறுதி" அல்ல, அதாவது file.txt கோப்பிற்குள் உள்ள உண்மையான உரையைப் பெற ஒரு படி நெருக்கமாக உள்ளோம்:

<template>
  <div>
    <button @click="fetchData">Fetch Data</button>
    <p v-if="data">{{ data }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      data: null,
    };
  },
  methods: {
    async fetchData() {
      const response = await fetch("file.txt");
      this.data = response;
    }
  }
};
</script>

file.txt கோப்பிற்குள் உள்ள உரையைப் பெற, பதிலில் text() முறையைப் பயன்படுத்த வேண்டும். text() முறை ஒரு வாக்குறுதி அடிப்படையிலான முறையாக இருப்பதால், அதன் முன்னால் await ஆபரேட்டரைப் பயன்படுத்த வேண்டும்.

இறுதியாக! file.txt கோப்பிற்குள் உள்ள உரையை fetch() முறையுடன் பெற நமக்குத் தேவையானது இப்போது உள்ளது:

<template>
  <div>
    <button @click="fetchData">Fetch Data</button>
    <p v-if="data">{{ data }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      data: null,
    };
  },
  methods: {
    async fetchData() {
      const response = await fetch("file.txt");
      this.data = await response.text();
    }
  }
};
</script>

ஒரு JSON கோப்பிலிருந்து தரவைப் பெறுதல்

முந்தைய எடுத்துக்காட்டில் .txt கோப்பிலிருந்து உரையைப் பெற்றோம். ஆனால் தரவைச் சேமிக்க பல வழிகள் உள்ளன, இப்போது ஒரு .json கோப்பிலிருந்து தகவலை எவ்வாறு பெறலாம் என்பதைப் பார்ப்போம்.

JSON என்பது ஒரு பொதுவான கோப்பு வடிவமாகும், இது தரவு உரையாகச் சேமிக்கப்படுவதால் மனிதர்களுக்கு படிக்க எளிதானது, மேலும் JSON வடிவமைப்பு நிரலாக்க மொழிகளால் பரவலாக ஆதரிக்கப்படுகிறது, எனவே நாம் எடுத்துக்காட்டாக, ஒரு .json கோப்பிலிருந்து எந்தத் தரவைப் பிரித்தெடுக்க வேண்டும் என்பதைக் குறிப்பிடலாம்.

.json கோப்பிலிருந்து தரவைப் படிக்க, மேலே உள்ள எடுத்துக்காட்டுக்கு நாம் செய்ய வேண்டிய ஒரே மாற்றம் .json கோப்பைப் பெறுவது மற்றும் பதிலில் text() முறைக்குப் பதிலாக json() முறையைப் பயன்படுத்துவதாகும்.

json() முறை HTTP கோரிக்கையிலிருந்து பதிலைப் படித்து ஒரு JavaScript பொருளை வழங்குகிறது.

<template>
  <div>
    <button @click="fetchData">Fetch Data</button>
    <pre v-if="data">{{ data }}</pre>
  </div>
</template>

<script>
export default {
  data() {
    return {
      data: null,
    };
  },
  methods: {
    async fetchData() {
      const response = await fetch("bigLandMammals.json");
      this.data = await response.json();
    }
  }
};
</script>

json() முறையின் விளைவு ஒரு JavaScript பொருளாக இருப்பதால், bigLandMammals.json கோப்பிலிருந்து ஒரு சீரற்ற விலங்கைக் காட்ட மேலே உள்ள எடுத்துக்காட்டை மாற்றலாம்:

<template>
  <p>Try clicking the button more than once to see new animals picked randomly.</p>
  <button @click="fetchData">Fetch Data</button>
  <div v-if="randomMammal">
    <h2>{{ randomMammal.name }}</h2>
    <p>Max weight: {{ randomMammal.maxWeight }} kg</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      randomMammal: null
    };
  },
  methods: {
    async fetchData() {
      const response = await fetch("bigLandMammals.json");
      const data = await response.json();
      const randIndex = Math.floor(Math.random()*data.results.length);
      this.randomMammal = data.results[randIndex];
    }
  }
};
</script>

நாம் இணைத்து பயன்படுத்தக்கூடிய பல சுவாரஸ்யமான இலவச APIகள் உள்ளன, வானிலை தரவு, பங்குச் சந்தை தரவு போன்றவற்றைப் பெற.

Vue HTTP Requests பயிற்சி

Vue இல் HTTP கோரிக்கைகள் பற்றிய உங்கள் புரிதலை சோதிக்க இந்த பயிற்சியை முடிக்கவும்.

❓ Vue.js இல் HTTP GET கோரிக்கையை செய்ய fetch() முறையைப் பயன்படுத்தும் போது, சரியான async/await தொடரியல் எது?

async fetchData() { const data = fetch("api/data"); }
✗ தவறு! await ஆபரேட்டர் இல்லாமல் fetch() ஒரு Promise மட்டுமே வழங்கும்
fetchData() { const data = await fetch("api/data"); }
✗ தவறு! await பயன்படுத்தப்படும் முறை async ஆக அறிவிக்கப்பட வேண்டும்
async fetchData() { const response = await fetch("api/data"); const data = await response.json(); }
✓ சரி! async முறை, fetch() க்கு முன்னால் await, மற்றும் response.json() க்கு முன்னால் await ஆகியவை சரியான தொடரியல்
async fetchData() { const data = fetch("api/data").json(); }
✗ தவறு! json() முறையை நேரடியாக Promise இல் அழைக்க முடியாது, முதலில் பதிலைப் பெற வேண்டும்

Vue இல் 'axios' நூலகத்துடன் HTTP கோரிக்கை

'axios' JavaScript நூலகமும் HTTP கோரிக்கைகளைச் செய்ய அனுமதிக்கிறது.

உங்கள் சொந்த கணினியில் எடுத்துக்காட்டை உருவாக்கவும் இயக்கவும், முதலில் உங்கள் திட்ட கோப்புறையில் terminal ஐப் பயன்படுத்தி 'axios' நூலகத்தை நிறுவ வேண்டும்:

npm install axios

ஒரு சீரற்ற பயனரைப் பெற Vue இல் 'axios' நூலகத்தைப் பயன்படுத்துவது எப்படி:

<template>
  <h1>Example</h1>
  <p>Click the button to fetch data with an HTTP request.</p>
  <button @click="fetchData">Fetch data</button>
  <div v-if="data" id="dataDiv">
    <img :src="data.data.avatar" alt="avatar">
    <pre>{{ data.data.first_name + " " + data.data.last_name }}</pre>
    <p>"{{ data.data.employment.title }}"</p>
  </div>
</template>

<script>
  import axios from 'axios'

  export default {
    data() {
      return {
        data: null,
      };
    },
    methods: {
      async fetchData() {      
        this.data = await axios.get("https://random-data-api.com/api/v2/users");
      }
    }
  };
</script>

⚡ fetch() vs axios:

fetch() என்பது நவீன உலாவிகளில் உள்ளிடப்பட்ட API ஆகும், அதே நேரத்தில் axios என்பது ஒரு மூன்றாம் தரப்பு நூலகமாகும். Axios தானாகவே JSON ஐ பார்க்கிங் செய்கிறது, பிழைக் கையாளுதலைக் கொண்டுள்ளது மற்றும் பின்னோக்கிய இணக்கத்தன்மையைக் கொண்டுள்ளது, அதே நேரத்தில் fetch() க்கு கூடுதல் கட்டமைப்பு தேவைப்படுகிறது.

HTTP முறைகள் ஒப்பீடு

முறை விளக்கம் பொதுவான பயன்பாடு
GET தரவை மீட்டெடுக்கிறது (படிக்க மட்டும்) பக்கங்கள், படங்கள், API தரவு ஏற்றுதல்
POST தரவைச் சமர்ப்பிக்கிறது/உருவாக்குகிறது படிவங்கள் சமர்ப்பிப்பு, புதிய உள்ளீடுகளை உருவாக்குதல்
PUT தரவை முழுமையாக மாற்றியமைக்கிறது முழு வளத்தையும் புதுப்பித்தல்
PATCH தரவை பகுதியாக மாற்றியமைக்கிறது குறிப்பிட்ட புலங்களைப் புதுப்பித்தல்
DELETE தரவை நீக்குகிறது வளங்களை நீக்குதல்
