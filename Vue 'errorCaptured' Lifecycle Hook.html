Vue 'errorCaptured' Lifecycle Hook

Vue errorCaptured லைஃப்சைக்கிள் ஹுக் பற்றி அறிக

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

errorCaptured லைஃப்சைக்கிள் ஹுக் பயன்படுத்தி child கம்போனெண்டிலிருந்து ஒரு error ஐ பிடித்து பயனருக்கு alert உருவாக்க.

<script>
export default {
  errorCaptured() {
    alert("An error occurred");
  }
}
</script>

இது எவ்வாறு செயல்படுகிறது என்பதைப் பார்க்க "Try it Yourself" பொத்தானைக் கிளிக் செய்யவும்.

மேலும் எடுத்துக்காட்டுகளை கீழே பார்க்கவும்.

வரையறை மற்றும் பயன்பாடு

errorCaptured லைஃப்சைக்கிள் ஹுக் child/descendant கம்போனெண்டில் error நடக்கும் போது அழைக்கப்படுகிறது.

இந்த ஹுக் error handling, logging, அல்லது பயனருக்கு error காட்ட பயன்படுத்தப்படலாம்.

⚠️ முக்கியமான குறிப்பு:

errorCaptured ஹுக் பயன்படுத்தும் போது, error வரும் கம்போனெண்டின் render ஐ trigger செய்யாமல் பார்த்துக்கொள்ள வேண்டும், ஏனெனில் அது பெரும்பாலும் ஒரு முடிவற்ற லூப்பை உருவாக்கும்.

error பற்றிய தகவல்கள் errorCaptured() செயல்பாட்டில் மூன்று arguments ஆக நமக்குக் கிடைக்கின்றன:

  1. error
  2. error ஐ trigger செய்த கம்போனெண்ட்
  3. error source type

error நடக்கும் போது default behavior என்பது, error நடந்த கம்போனெண்டிலிருந்து propagate, அல்லது 'bubble up' ஆகும். ஒரு கம்போனெண்டில் error நடந்தால், அது parent கம்போனெண்டுக்கு மேலே செல்லும், மேலும் தொடர்ந்து மேலே செல்லும், இறுதியில் கன்சோலில் error message ஆக முடியும்.

errorCaptured() செயல்பாட்டிற்குள் return false; ஐ இயக்குவதன் மூலம், error parent கம்போனெண்டில் முடியாது (propagate நிறுத்தப்படும்), மேலும் error கன்சோலில் error message ஆகவும் முடியாது.

error handling app.config.errorHandler செயல்பாட்டைப் பயன்படுத்தியும் அமைக்கப்படலாம்.

மேலும் எடுத்துக்காட்டுகள்

எடுத்துக்காட்டு 1

errorCaptured லைஃப்சைக்கிள் ஹுக் பயன்படுத்தி error ஐ பிடித்து error பற்றிய தகவல்களை கன்சோலில் எழுத.

App.vue:

<template>
  <h1>The 'errorCaptured' Lifecycle Hook</h1>
  <p>Whenever there is an error in a child component, the errorCaptured() function is called on the parent.</p>
  <p>Open the browser console to see the captured error details.</p>
  <div>
    <comp-one></comp-one>
  </div>
</template>

<script>
export default {
  errorCaptured(error,compInst,errorInfo) {
    console.log("error: ", error);
    console.log("compInst: ", compInst);
    console.log("errorInfo: ", errorInfo);
  }
}
</script>

<style>
#app > div {
  border: dashed black 1px;
  border-radius: 10px;
  padding: 10px;
  margin-top: 10px;
  background-color: lightgreen;
}
</style>

ComOne.vue:

<template>
  <h2>Component</h2>
  <p>This is the component</p>
  <button @click="generateError">Generate Error</button>
</template>

<script>
export default {
  methods: {
    generateError() {
      this.$refs.objEl.innerHTML = "hi";
    }
  }
}
</script>

எடுத்துக்காட்டு 2

errorCaptured லைஃப்சைக்கிள் ஹுக் return false; உடன் பயன்படுத்தி error propagate ஆகாமல் தடுக்க.

App.vue:

<template>
  <h1>The 'errorCaptured' Lifecycle Hook</h1>
  <p>Whenever there is an error in a child component, the errorCaptured() function is called on the parent.</p>
  <p>Open the browser console to see the captured error details.</p>
  <div>
    <comp-one></comp-one>
  </div>
</template>

<script>
export default {
  errorCaptured(error,compInst,errorInfo) {
    console.log("error: ", error);
    console.log("compInst: ", compInst);
    console.log("errorInfo: ", errorInfo);
    return false;
  }
}
</script>

}
</style>

Vue பயிற்சிகள்

இந்த டுடோரியலில் உள்ள பல அத்தியாயங்கள் உங்கள் அறிவு நிலையைச் சரிபார்க்கக்கூடிய பயிற்சியுடன் முடிகின்றன.

❓ errorCaptured லைஃப்சைக்கிள் ஹுக் பற்றிய சரியான கூற்று எது?

errorCaptured ஹுக் child கம்போனெண்டில் error நடக்கும் போது parent கம்போனெண்டில் அழைக்கப்படுகிறது
✓ சரி! errorCaptured ஹுக் child அல்லது descendant கம்போனெண்டில் error நடக்கும் போது parent கம்போனெண்டில் அழைக்கப்படுகிறது. இது error propagation chain இல் வேலை செய்கிறது
errorCaptured ஹுக்கில் error ஐ முழுமையாக மறைக்க முடியும்
✗ தவறு! errorCaptured ஹுக்கில் return false; உடன் error propagation ஐ தடுக்க முடியும், ஆனால் error ஐ முழுமையாக மறைக்க முடியாது. error ஏற்கனவே நடந்து விட்டது
errorCaptured ஹுக்கில் render செய்யும் குறியீட்டை இயக்கலாம்
✗ தவறு! errorCaptured ஹுக்கில் render ஐ trigger செய்யும் குறியீட்டை இயக்கக்கூடாது, ஏனெனில் அது infinite loop ஏற்படுத்தும். error ஐ handle செய்யும் குறியீடு மட்டுமே இயக்க வேண்டும்
errorCaptured ஹுக்கு arguments இல்லை
✗ தவறு! errorCaptured ஹுக்கு மூன்று arguments உள்ளன: 1. error object, 2. error ஐ trigger செய்த கம்போனெண்ட், 3. error source type. இவை error பற்றிய முழு தகவல்களை வழங்குகின்றன
