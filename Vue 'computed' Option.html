Vue 'computed' விருப்பத்தைக் கற்றுக்கொள்ளுங்கள்

Vue 'computed' Option

🎯 Example

சரியான பொத்தான் உரையைக் காட்ட computed விருப்பத்திற்குள் ஒரு கம்ப்யூட்டட் ப்ராபர்ட்டியைப் பயன்படுத்துதல்.

export default {
  data() {
    return {
      msg: 'Hello World!',
      showMsg: false
    };
  },
  computed: {
    btnText() {
      if( this.showMsg ) {
        return 'Hide'
      }
      else {
        return 'Show'
      }
    }
  }
};

Definition and Usage

computed விருப்பம் என்பது Vue உதாரணத்தில் அறிவிக்கப்பட்ட அனைத்து கம்ப்யூட்டட் ப்ராபர்ட்டீஸையும் கொண்ட ஒரு பொருளாகும்.

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் பொதுவாக படிக்க-மட்டும் (மேலே உள்ள உதாரணத்தைப் பார்க்கவும்), ஆனால் ஒரு get மற்றும் set செயல்பாடு இரண்டையும் கொண்ட ஒரு பொருளாக ஒரு கம்ப்யூட்டட் ப்ராபர்ட்டியை வரையறுக்க முடியும், அதாவது கம்ப்யூட்டட் ப்ராபர்ட்டியும் எழுதப்பட முடியும்.

⚠️ குறிப்பு:

கம்ப்யூட்டட் ப்ராபர்ட்டீஸை அறிவிக்கும் போது அம்புக்குறி செயல்பாடுகள் தவிர்க்கப்பட வேண்டும், ஏனெனில் this முக்கிய சொல்லைப் பயன்படுத்தி அத்தகைய செயல்பாட்டிற்குள் இருந்து Vue உதாரணத்தை அணுக முடியாது.

Vue Exercises

❓ Vue.js இல், கம்ப்யூட்டட் ப்ராபர்ட்டீஸின் முக்கிய நன்மை என்ன?

அவை அவற்றின் சார்புகள் மாறும்போது மட்டுமே மறுகணக்கிடப்படுகின்றன மற்றும் முடிவுகளைக் கேஷ் செய்கின்றன
✓ சரி! கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் அவற்றின் சார்புகள் மாறும்போது மட்டுமே மறுகணக்கிடப்படுகின்றன, மேலும் அவை முடிவுகளைக் கேஷ் செய்கின்றன. இது சிக்கலான கணக்கீடுகளுக்கு சிறந்த செயல்திறனை வழங்குகிறது
அவை முறைகளை விட வேகமாக இயங்கும், அதனால்தான் அவற்றை எப்போதும் பயன்படுத்த வேண்டும்
✗ தவறு! கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் எப்போதும் முறைகளை விட வேகமாக இல்லை. அவை சார்புகள் அடிக்கடி மாறாத இடத்தில் செயல்திறனுக்காக வடிவமைக்கப்பட்டுள்ளன. நிகழ்வு கையாளுதலுக்கு முறைகள் இன்னும் சரியானவை
அவை தரவை மாற்றலாம், அதனால்தான் அவை முறைகளை விட சக்திவாய்ந்தவை
✗ தவறு! கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் பொதுவாக படிக்க-மட்டும் மற்றும் தரவை மாற்றக்கூடாது. அவை மதிப்புகளைக் கணக்கிட்டு திருப்பி அனுப்பும் வகையில் வடிவமைக்கப்பட்டுள்ளன, அசைவு விளைவுகள் இல்லாமல்
அவை வாதங்களை ஏற்க முடியாது, அதனால்தான் அவை எளிமையானவை
✗ தவறு! கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் வாதங்களை ஏற்காது என்பது உண்மை, ஆனால் இது அவற்றின் முக்கிய நன்மை அல்ல. அவற்றின் முக்கிய நன்மை கேஷிங் வழிமுறை மற்றும் தானியங்கி மறுகணக்கீடு ஆகும்
