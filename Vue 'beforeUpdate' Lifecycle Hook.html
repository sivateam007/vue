Vue 'beforeUpdate' Lifecycle Hook

Vue beforeUpdate லைஃப்சைக்கிள் ஹுக் பற்றி அறிக

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

beforeUpdate லைஃப்சைக்கிள் ஹுக் பயன்படுத்தி எத்தனை முறை அப்டேட் டிரிகர் செய்யப்படுகிறது என எண்ண.

export default {
  data() {
    return {
      sliderVal: 50,
      renderCount: 0
    }
  },
  beforeUpdate() {
    this.renderCount++;
  }
}

இது எவ்வாறு செயல்படுகிறது என்பதைப் பார்க்க "Try it Yourself" பொத்தானைக் கிளிக் செய்யவும்.

வரையறை மற்றும் பயன்பாடு

beforeUpdate லைஃப்சைக்கிள் ஹுக் Vue இன் ரியாக்டிவ் சிஸ்டம் புதிய ரெண்டரிங் தேவைப்படும் மாற்றத்தை கண்டறிந்த பிறகு நடக்கும், ஆனால் அந்த ரெண்டரிங் நடப்பதற்கு முன்.

ஒரு முடிவற்ற லூப்பைத் தவிர்க்க நாம் எப்போதும் updated லைஃப்சைக்கிள் ஹுக்கிற்குப் பதிலாக beforeUpdate லைஃப்சைக்கிள் ஹுக் பயன்படுத்துவதைக் கருத்தில் கொள்ள வேண்டும்.

இந்த டுடோரியலில் உள்ள பல அத்தியாயங்கள் உங்கள் அறிவு நிலையைச் சரிபார்க்கக்கூடிய பயிற்சியுடன் முடிகின்றன.

❓ beforeUpdate லைஃப்சைக்கிள் ஹுக் பற்றிய சரியான கூற்று எது?

beforeUpdate ஹுக் ரெண்டரிங் நடப்பதற்கு முன் நடக்கும்
✓ சரி! beforeUpdate ஹுக் Vue இன் ரியாக்டிவ் சிஸ்டம் மாற்றத்தை கண்டறிந்த பிறகு, ஆனால் புதிய ரெண்டரிங் நடப்பதற்கு முன் நடக்கும்
beforeUpdate ஹுக்கில் டேட்டா மாற்றங்களை செய்யலாம், இது infinite loop ஏற்படாது
✗ தவறு! beforeUpdate ஹுக்கில் டேட்டா மாற்றங்களை செய்தால் infinite loop ஏற்படலாம். ரெண்டரிங் தேவைப்படும் மாற்றங்களைத் தவிர்க்கவும்
beforeUpdate ஹுக் கம்போனெண்ட் முதல் முறையாக உருவாக்கப்படும் போது நடக்கும்
✗ தவறு! beforeUpdate ஹுக் கம்போனெண்ட் முதல் முறையாக உருவாக்கப்படும் போது நடக்காது. இது ரெண்டரிங் தேவைப்படும் மாற்றம் கண்டறியப்பட்ட பிறகு மட்டுமே நடக்கும்
beforeUpdate ஹுக்கில் DOM எலிமெண்டுகள் இன்னும் புதுப்பிக்கப்படவில்லை
✗ தவறு! beforeUpdate ஹுக்கில் DOM எலிமெண்டுகள் ஏற்கனவே புதுப்பிக்கப்பட்டுள்ளன. beforeUpdate ஹுக் ரெண்டரிங் நடப்பதற்கு முன் நடக்கும், ஆனால் DOM புதுப்பித்தல் நடப்பதற்கு முன் அல்ல
