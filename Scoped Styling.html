Scoped Styling

Vue Scoped ஸ்டைலிங் பற்றி அறிக

Scoped ஸ்டைலிங்

ஒரு கம்போனெண்டில் <style> டேக்கிற்குள் வரையறுக்கப்பட்ட ஸ்டைலிங், அல்லது App.vue இல், உண்மையில் அனைத்து கம்போனெண்ட்களிலும் globally கிடைக்கும்.

ஸ்டைலிங் கம்போனெண்டுக்கு உள்ளூராக மட்டுமே வரையறுக்க, அந்த கம்போனெண்டில் scope attribute ஐப் பயன்படுத்தலாம்: <style scoped>

Global ஸ்டைலிங்

எந்த *.vue ஃபைலிலும் <style> டேக்கிற்குள் எழுதப்பட்ட CSS globally வேலை செய்கிறது.

இதன் பொருள், நாம் எடுத்துக்காட்டாக ஒரு *.vue ஃபைலில் <style> டேக்கிற்குள் <p> டேக்குகளுக்கு pink background color அமைத்தால், இது அந்த project இல் உள்ள அனைத்து *.vue ஃபைல்களிலும் உள்ள <p> டேக்குகளை பாதிக்கும்.

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

இந்த பயன்பாட்டில் நம்மிடம் மூன்று *.vue ஃபைல்கள் உள்ளன: App.vue, மற்றும் இரண்டு கம்போனெண்ட்கள் CompOne.vue மற்றும் CompTwo.vue.

CompOne.vue இல் உள்ள CSS ஸ்டைலிங் மூன்று *.vue ஃபைல்களிலும் உள்ள <p> டேக்குகளை பாதிக்கிறது:

<template>
  <p>This p-tag belongs to 'CompOne.vue'</p>
</template>

<script></script>

<style>
  p {
    background-color: pink;
    width: 150px;
  }
</style>

இது எவ்வாறு செயல்படுகிறது என்பதைப் பார்க்க "Try it Yourself" பொத்தானைக் கிளிக் செய்யவும்.

Scoped ஸ்டைலிங்

ஒரு கம்போனெண்டில் உள்ள ஸ்டைலிங் மற்ற கம்போனெண்ட்களில் உள்ள எலிமெண்ட்களின் ஸ்டைலிங்கை பாதிக்காமல் இருப்பதற்கு நாம் <style> டேக்கில் 'scoped' attribute ஐப் பயன்படுத்துகிறோம்:

எடுத்துக்காட்டு

CompOne.vue இல் உள்ள <style> டேக்கிற்கு scoped attribute கொடுக்கப்படுகிறது:

<template>
  <p>This p-tag belongs to 'CompOne.vue'</p>
</template>

<script></script>

<style scoped>
  p {
    background-color: pink;
    width: 150px;
  }
</style>

💡 Scoped ஸ்டைலிங் எவ்வாறு வேலை செய்கிறது:

Scoped attribute இணைக்கப்படும் போது, Vue தானாகவே ஒரு தனிப்பட்ட data attribute ஐ கம்போனெண்டின் எலிமெண்ட்களுக்குச் சேர்க்கிறது. CSS selectors இந்த data attribute உடன் தானாகவே update செய்யப்படுகின்றன, இதனால் ஸ்டைலிங் அந்த குறிப்பிட்ட கம்போனெண்டுக்கு மட்டுமே பயன்படுத்தப்படுகிறது.

Global vs Scoped ஸ்டைலிங் ஒப்பீடு

Global ஸ்டைலிங்

அனைத்து கம்போனெண்ட்களுக்கும் பொருந்தும்

Syntax: <style>

குறிப்பிட்ட கம்போனெண்டுக்கு ஸ்டைலிங் கட்டுப்படுத்த முடியாது

மிகவும் எளிமையானது, ஆனால் பராமரிக்க கடினமாக இருக்கலாம்

Scoped ஸ்டைலிங்

கம்போனெண்டுக்கு உள்ளூராக மட்டுமே பொருந்தும்

Syntax: <style scoped>

ஸ்டைலிங் மோதல்களை தடுக்கிறது

பெரிய ப்ராஜெக்ட்களுக்கு மிகவும் பயனுள்ளது

Vue பயிற்சிகள்

இந்த டுடோரியலில் உள்ள பல அத்தியாயங்கள் உங்கள் அறிவு நிலையைச் சரிபார்க்கக்கூடிய பயிற்சியுடன் முடிகின்றன.

❓ Scoped ஸ்டைலிங் பற்றிய சரியான கூற்று எது?

Scoped ஸ்டைலிங் அனைத்து கம்போனெண்ட்களுக்கும் பொருந்தும்
✗ தவறு! Scoped ஸ்டைலிங் குறிப்பிட்ட கம்போனெண்டுக்கு மட்டுமே பொருந்தும். இது ஸ்டைலிங் மோதல்களை தடுக்கிறது
Scoped ஸ்டைலிங் கம்போனெண்டுக்கு உள்ளூராக ஸ்டைலிங் கட்டுப்படுத்த பயன்படுகிறது
✓ சரி! Scoped ஸ்டைலிங் <style scoped> attribute பயன்படுத்தி கம்போனெண்டுக்கு உள்ளூராக ஸ்டைலிங் கட்டுப்படுத்த பயன்படுகிறது. இது CSS ஐ தானாகவே தனித்துவமான selectors உடன் scope செய்கிறது
Scoped ஸ்டைலிங் Vue 2 இல் கிடைக்காது
✗ தவறு! Scoped ஸ்டைலிங் Vue 2 மற்றும் Vue 3 இரண்டிலும் கிடைக்கும். இது Vue இன் built-in feature ஆகும்
Scoped ஸ்டைலிங் child கம்போனெண்ட்களுக்கும் பொருந்தும்
✗ தவறு! Scoped ஸ்டைலிங் child கம்போனெண்ட்களுக்கு பொருந்தாது. இது குறிப்பிட்ட கம்போனெண்டுக்கு மட்டுமே உள்ளது. Child கம்போனெண்ட்கள் தங்கள் சொந்த scoped ஸ்டைலிங் கொண்டிருக்க வேண்டும்

நடைமுறை உதவிக்குறிப்புகள்

பெரிய ப்ராஜெக்ட்களில் எப்போதும் scoped ஸ்டைலிங் பயன்படுத்தவும்
Global ஸ்டைலிங் முக்கியமான விளைவுகள் கொண்ட ப்ராஜெக்ட்களில் தவிர்க்கவும்
Scoped மற்றும் global ஸ்டைலிங் கலந்து பயன்படுத்தலாம்
Scoped ஸ்டைலிங் CSS specificity issues ஐ குறைக்கிறது

⚠️ கவனத்திற்கு:

Scoped ஸ்டைலிங் CSS selectors ஐ தானாகவே மாற்றுகிறது, எனவே சில CSS frameworks அல்லது libraries உடன் சிக்கல்கள் இருக்கலாம். எப்போதும் உங்கள் பயன்பாடு சரியாக வேலை செய்கிறதா என சோதிக்கவும்.
