Vue Fallthrough Attributes

Vue பால்த்ரூ அட்ரிப்யூட்ஸ் பற்றி அறிக

Vue பால்த்ரூ அட்ரிப்யூட்ஸ்

ஒரு கம்போனெண்டை props ஆக அறிவிக்கப்படாத attributes உடன் அழைக்க முடியும், மேலும் அவை வெறுமனே கம்போனெண்டில் உள்ள root element க்கு fall through ஆகிவிடும்.

பால்த்ரூ அட்ரிப்யூட்ஸ் மூலம் நீங்கள் கம்போனெண்ட் உருவாக்கப்படும் parent இலிருந்து சிறந்த overview ஐப் பெறுகிறீர்கள், மேலும் attribute ஐ prop ஆக அறிவிக்க தேவையில்லாததால் எங்கள் குறியீடு எளிமையாகிறது.

fall through ஆகப் பயன்படுத்தப்படும் பொதுவான attributes கள் class, style மற்றும் v-on ஆகும்.

பால்த்ரூ அட்ரிப்யூட்ஸ்

உதாரணமாக, கம்போனெண்டிற்குள் styling மறைந்து விடுவதை விட parent இலிருந்து கம்போனெண்ட் styling ஐ கட்டுப்படுத்துவது நல்லது.

Vue இல் ஒரு அடிப்படை to-do list உருவாக்கி, style attribute செய்ய வேண்டிய விஷயங்களைக் குறிக்கும் கம்போனெண்ட்களுக்கு எவ்வாறு fall through ஆகிறது என்பதைப் பார்ப்போம்.

எனவே, எங்கள் App.vue செய்ய வேண்டிய விஷயங்களின் பட்டியலைக் கொண்டிருக்க வேண்டும், மேலும் புதிய விஷயங்களைச் சேர்க்க element மற்றும்

App.vue:

<template>
  <h3>Todo List</h3>  
  <ul>
    <todo-item
      v-for="x in items"
      :key="x"
      :item-name="x"
    />
  </ul>
  <input v-model="newItem">
  <button @click="addItem">Add</button>
</template>

<script>
  export default {
    data() {
      return {
        newItem: '',
        items: ['Buy apples','Make pizza','Mow the lawn']
      };
    },
    methods: {
      addItem() {
        this.items.push(this.newItem),
        this.newItem = '';
      }
    }
  }
</script>

TodoItem.vue:

என்ன செய்ய வேண்டும் என்பதை விவரிக்கும் prop ஆகப் பெறுகிறது:

<template>
  <li>{{ itemName }}</li>
</template>

<script>
  export default {
    props: ['itemName']
  }
</script>

main.js:

import { createApp } from 'vue'
  
import App from './App.vue'
import TodoItem from './components/TodoItem.vue'

const app = createApp(App)
app.component('todo-item', TodoItem)
app.mount('#app')

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

கம்போனெண்டிற்குள் உள்ள <li> elements க்கு styling கொடுக்க App.vue இலிருந்து:

<template>
  <h3>Todo List</h3>
  <ul>
    <todo-item
      v-for="x in items"
      :key="x"
      :item-name="x"
      style="background-color: lightgreen;"
    />
  </ul>
  <input v-model="newItem">
  <button @click="addItem">Add</button>
</template>

🔍 சரிபார்த்தல்:

style attribute உண்மையில் fall through ஆகிவிட்டதா என உறுதிப்படுத்த, நமது to-do list இல் உள்ள <li> element மீது browser இல் வலது க்ளிக் செய்து 'Inspect' தேர்வு செய்யலாம், மேலும் style attribute இப்போது <li> element இல் உள்ளதைக் காணலாம்.

25.png: style attribute <li> element இல் உள்ளது என்பதைக் காட்டும் படம்

'class' மற்றும் 'style' Attributes ஐ Merge செய்தல்

'class' அல்லது 'style' attributes ஏற்கனவே அமைக்கப்பட்டிருந்தால், மற்றும் 'class' அல்லது 'style' attributes parent இலிருந்து பால்த்ரூ அட்ரிப்யூட்ஸ் ஆகவும் வந்தால், attributes merge செய்யப்படும்.

எடுத்துக்காட்டு

Parent இலிருந்து இருக்கும் styling க்கு கூடுதலாக, TodoItem.vue கம்போனெண்டிற்குள் <li> elements க்கு margin சேர்க்கிறோம்:

<template>
  <li style="margin: 5px 0;">{{ itemName }}</li>
</template>

<script>
  export default {
    props: ['itemName']
  }
</script>

🔍 சரிபார்த்தல்:

Browser இல் <li> element மீது வலது க்ளிக் செய்தால், attributes merge செய்யப்பட்டிருப்பதைக் காணலாம். Margin நேரடியாக கம்போனெண்டிற்குள் உள்ள <li> element இல் அமைக்கப்பட்டுள்ளது, மேலும் parent இலிருந்து fall through ஆகும் background-color உடன் merge செய்யப்பட்டுள்ளது.

26.png: attributes merge செய்யப்பட்டதைக் காட்டும் படம்

$attrs

கம்போனெண்டின் root level இல் ஒன்றுக்கு மேற்பட்ட element கள் இருந்தால், attributes எந்த element க்கு fall through ஆக வேண்டும் என்பது இனி தெளிவாக இருக்காது.

எந்த root element பால்த்ரூ அட்ரிப்யூட்ஸ் பெறுகிறது என வரையறுக்க, built-in $attrs object உடன் element ஐ குறிக்கலாம், இது போல்:

எடுத்துக்காட்டு

TodoItem.vue:

<template>
  <div class="pinkBall"></div>
  <li v-bind="$attrs">{{ itemName }}</li>
  <div class="pinkBall"></div>
</template>

Vue பயிற்சிகள்

பயிற்சிகளுடன் உங்களை சோதிக்கவும்

✏️ பயிற்சி:

'fish-type' கம்போனெண்டின் root element 'blue' CSS class க்கு சொந்தமானதாக அமைக்கவும் (ஒரு பால்த்ரூ அட்ரிப்யூட் உருவாக்கவும்).

<fish-type 
  />
class: "blue"
✗ தவறு! class attribute இல் colon (:) தேவையில்லை
:class="blue"
✗ தவறு! இது 'blue' என்ற string அல்ல, இது 'blue' variable ஐ reference செய்கிறது
class="blue"
✓ சரி! இது 'blue' CSS class ஐ பால்த்ரூ அட்ரிப்யூட் ஆக அமைக்கிறது, இது கம்போனெண்டின் root element க்கு fall through ஆகிவிடும்
v-bind:class="'blue'"
✗ தவறு! இது வேலை செய்யும், ஆனால் இது எளிய fallthrough attribute க்கு மிகவும் சிக்கலானது
