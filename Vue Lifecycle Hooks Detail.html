Vue வாழ்க்கைச் சுழற்சி கொக்கிகள்

Vue இல் வாழ்க்கைச் சுழற்சி கொக்கிகள் என்பது ஒரு கூறின் வாழ்க்கைச் சுழற்சியில் குறிப்பிட்ட நிலைகள் ஆகும், அங்கு விஷயங்களைச் செய்ய குறியீட்டைச் சேர்க்கலாம்.

வாழ்க்கைச் சுழற்சி கொக்கிகள் கண்ணோட்டம்

ஒரு கூறு அதன் வாழ்க்கைச் சுழற்சியில் ஒரு புதிய நிலையை அடையும் ஒவ்வொரு முறையும், ஒரு குறிப்பிட்ட செயல்பாடு இயங்குகிறது, மேலும் அந்த செயல்பாட்டில் குறியீட்டைச் சேர்க்கலாம். அத்தகைய செயல்பாடுகள் வாழ்க்கைச் சுழற்சி கொக்கிகள் என்று அழைக்கப்படுகின்றன, ஏனெனில் அந்த நிலையில் எங்கள் குறியீட்டை "கொக்கி" செய்யலாம்.

'beforeCreate' கொக்கி

beforeCreate வாழ்க்கைச் சுழற்சி கொக்கி கூறு துவக்கப்படுவதற்கு முன் நடக்கிறது, எனவே Vue கூறின் தரவு, கணக்கிடப்பட்ட பண்புகள், முறைகள் மற்றும் நிகழ்வு கேட்பவர்களை அமைக்கும் முன் இது.

எடுத்துக்காட்டாக உலகளாவிய நிகழ்வு கேட்பவரை அமைக்க beforeCreate கொக்கியைப் பயன்படுத்தலாம், ஆனால் beforeCreate வாழ்க்கைச் சுழற்சி கொக்கியிலிருந்து கூறுக்குச் சொந்தமான உறுப்புகளை அணுக முயற்சிக்க வேண்டாம், ஏனெனில் அவை இந்த கட்டத்தில் இன்னும் உருவாக்கப்படவில்லை.

மேலும், beforeCreate வாழ்க்கைச் சுழற்சி கொக்கியிலிருந்து DOM உறுப்புகளை அணுக முயற்சிப்பதில் அர்த்தமில்லை, ஏனெனில் அவை கூறு ஏற்றப்பட்ட பிறகு வரை உருவாக்கப்படாது.

எடுத்துக்காட்டு
CompOne.vue
<template>
    <h2>Component</h2>
    <p>This is the component</p>
    <p id="pResult">{{ text }}</p>
</template>

<script>
export default {
    data() {
        return {
            text: '...'
        }
    },
    beforeCreate() {
        this.text = 'initial text'; // இந்த வரியால் விளைவு இல்லை
        console.log("beforeCreate: கூறு இன்னும் உருவாக்கப்படவில்லை.");
    }
}
</script>
App.vue
<template>
    <h1>The 'beforeCreate' Lifecycle Hook</h1>
    <p>We can see the console.log() message...</p>
    <button @click="this.activeComp = !this.activeComp">
        Add/Remove Component
    </button>
    <div>
        <comp-one v-if="activeComp"></comp-one>
    </div>
</template>

<script>
export default {
    data() {
        return {
            activeComp: false
        }
    }
}
</script>

குறிப்பு: மேலே உள்ள எடுத்துக்காட்டில், CompOne.vue இல் வரி 15 க்கு விளைவு இல்லை, ஏனெனில் அந்த வரியில் Vue தரவு பண்புக்குள் உள்ள உரையை மாற்ற முயற்சிக்கிறோம், ஆனால் Vue தரவு பண்பு உண்மையில் இன்னும் உருவாக்கப்படவில்லை. மேலும், console.log() அழைப்பின் முடிவைக் காண உலாவி கன்சோலைத் திறக்க நினைவில் கொள்ளுங்கள்.

'created' கொக்கி

created வாழ்க்கைச் சுழற்சி கொக்கி கூறு துவக்கப்பட்ட பிறகு நடக்கிறது, எனவே Vue ஏற்கனவே கூறின் தரவு, கணக்கிடப்பட்ட பண்புகள், முறைகள் மற்றும் நிகழ்வு கேட்பவர்களை அமைத்துள்ளது.

created வாழ்க்கைச் சுழற்சி கொக்கியிலிருந்து DOM உறுப்புகளை அணுக முயற்சிக்க வேண்டாம், ஏனெனில் கூறு ஏற்றப்படும் வரை DOM உறுப்புகள் அணுக முடியாது.

HTTP கோரிக்கைகளுடன் தரவை எடுக்க அல்லது ஆரம்ப தரவு மதிப்புகளை அமைக்க created வாழ்க்கைச் சுழற்சி கொக்கியைப் பயன்படுத்தலாம்.

'beforeMount' கொக்கி

beforeMount வாழ்க்கைச் சுழற்சி கொக்கி கூறு ஏற்றப்படுவதற்கு சற்று முன் நடக்கிறது, எனவே கூறு DOM க்கு சேர்க்கப்படுவதற்கு சற்று முன்.

beforeMount வாழ்க்கைச் சுழற்சி கொக்கியிலிருந்து DOM உறுப்புகளை அணுக முயற்சிக்க வேண்டாம், ஏனெனில் கூறு ஏற்றப்படும் வரை DOM உறுப்புகள் அணுக முடியாது.

'mounted' கொக்கி

ஒரு கூறு DOM மரத்தில் சேர்க்கப்பட்ட உடனேயே, mounted() செயல்பாடு அழைக்கப்படுகிறது, மேலும் அந்த கட்டத்தில் எங்கள் குறியீட்டைச் சேர்க்கலாம்.

கூறுக்குச் சொந்தமான DOM உறுப்புகள் தொடர்பான விஷயங்களைச் செய்ய இது முதல் வாய்ப்பு.

எடுத்துக்காட்டு
<template>
    <h2>Form Component</h2>
    <p>When this component is added to the DOM tree...</p>
    <form @submit.prevent>
        <label>
            <p>
                Name: <br>
                <input type="text" ref="inpName">
            </p>
        </label>
        <button>Submit</button>
    </form>
</template>

<script>
export default {
    mounted() {
        this.$refs.inpName.focus();
    }
}
</script>

💻 உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

குறிப்பு: mounted கட்டம் கூறு DOM க்கு சேர்க்கப்பட்ட பிறகு நடக்கிறது, ஆனால் மேலே உள்ள எடுத்துக்காட்டில் கூறைக் காணும் முன் alert() தெரியும். இதற்கான காரணம், முதலில் கூறு DOM க்கு சேர்க்கப்படுகிறது, ஆனால் உலாவி கூறைத் திரையில் ரெண்டர் செய்வதற்கு முன், mounted கட்டம் நடக்கிறது மற்றும் alert() தெரியும் மற்றும் கூறை ரெண்டர் செய்வதை உலாவி இடைநிறுத்துகிறது.

'beforeUpdate' கொக்கி

beforeUpdate வாழ்க்கைச் சுழற்சி கொக்கி எங்கள் கூறின் தரவில் மாற்றம் ஏற்படும் போதெல்லாம் அழைக்கப்படுகிறது, ஆனால் புதுப்பிப்பு திரையில் ரெண்டர் செய்யப்படுவதற்கு முன். beforeUpdate வாழ்க்கைச் சுழற்சி கொக்கி updated வாழ்க்கைச் சுழற்சி கொக்கிக்கு சற்று முன் நடக்கிறது.

beforeUpdate கொக்கியைப் பற்றிய சிறப்பு விஷயம் என்னவென்றால், புதிய புதுப்பிப்பைத் தூண்டாமல் நாம் பயன்பாட்டில் மாற்றங்களைச் செய்யலாம், எனவே இல்லையெனில் முடிவில்லாத வளையத்தைத் தவிர்க்கலாம்.

'updated' கொக்கி

updated வாழ்க்கைச் சுழற்சி கொக்கி எங்கள் கூறு அதன் DOM மரத்தைப் புதுப்பித்த பிறகு அழைக்கப்படுகிறது.

⚠️ எச்சரிக்கை:

updated வாழ்க்கைச் சுழற்சி கொக்கி அழைக்கப்படும் போது பக்கத்தை மாற்றுவதில் கவனமாக இருக்க வேண்டும், ஏனெனில் பக்கம் மீண்டும் மீண்டும் புதுப்பிக்கப்படும், முடிவில்லாத வளையத்தை உருவாக்கும்.

எச்சரிக்கை எடுத்துக்காட்டு
<template>
    <h1>The 'updated' Lifecycle Hook</h1>
    <button @click="this.activeComp = !this.activeComp">
        Add/Remove Component
    </button>
    <div>
        <comp-one v-if="activeComp"></comp-one>
    </div>
    <div>{{ text }}</div>
</template>

<script>
export default {
    data() {
        return {
            activeComp: true,
            text: "Hello, "
        }
    },
    updated() {
        this.text += "hi, "; // ⚠️ முடிவில்லாத வளையத்தை உருவாக்குகிறது!
    }
}
</script>

'beforeUnmount' கொக்கி

beforeUnmount வாழ்க்கைச் சுழற்சி கொக்கி ஒரு கூறு DOM இலிருந்து நீக்கப்படுவதற்கு சற்று முன் அழைக்கப்படுகிறது.

beforeUnmount கொக்கியில் DOM இல் உள்ள கூறு உறுப்புகளை இன்னும் அணுக முடியும் என்பதை கீழே உள்ள எடுத்துக்காட்டில் காணலாம்.

'unmounted' கொக்கி

unmounted வாழ்க்கைச் சுழற்சி கொக்கி ஒரு கூறு DOM இலிருந்து நீக்கப்பட்ட பிறகு அழைக்கப்படுகிறது.

இந்த கொக்கியை எடுத்துக்காட்டாக நிகழ்வு கேட்பவர்களை அகற்ற அல்லது நேரமக்கிகள் அல்லது இடைவெளிகளை ரத்து செய்ய பயன்படுத்தலாம்.

குறிப்பு: unmounted கட்டம் கூறு DOM இலிருந்து நீக்கப்பட்ட பிறகு நடக்கிறது, ஆனால் மேலே உள்ள எடுத்துக்காட்டில் கூறு மறைவதற்கு முன் alert() தெரியும். இதற்கான காரணம், முதலில் கூறு DOM இலிருந்து நீக்கப்படுகிறது, ஆனால் உலாவி கூறின் நீக்கத்தைத் திரையில் ரெண்டர் செய்வதற்கு முன், unmounted கட்டம் நடக்கிறது மற்றும் alert() தெரியும் மற்றும் கூறைக் காண்பிக்க உலாவி இடைநிறுத்துகிறது.
