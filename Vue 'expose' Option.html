Vue 'expose' Option

🏁 உதவிக்குறிப்பு:

உங்கள் Vue சேவையகத்தைப் பெறுங்கள்

பெற்றோர் கூறு பயன்படுத்துவதற்கு ஒரு முறையைக் கிடைக்கச் செய்ய expose விருப்பத்தைப் பயன்படுத்துதல்.

Example

export default {
  expose: ['createMessage'],
  data() {
    return {
      message: ' '
    }
  },
  methods: {
    createMessage(msg) {
      this.message += msg + ' '
    }
  }
}

கீழே மேலும் எடுத்துக்காட்டுகளைப் பார்க்கவும்

வரையறை மற்றும் பயன்பாடு

expose விருப்பம் டெம்ப்ளேட் refs மூலம் பெற்றோர் கூறுக்குக் கிடைக்கும் பண்புகளைப் பட்டியலிட பயன்படுகிறது.

இயல்பாக, அனைத்து குழந்தை கூறு பண்புகளும் டெம்ப்ளேட் refs பயன்பாட்டின் மூலம் பெற்றோர் கூறுக்குக் கிடைக்கும்.

இதன் பொருள் குழந்தை கூறுக்கு expose விருப்பம் இல்லையென்றால், மற்றும் பெற்றோர் கூறு குழந்தை கூறில் உள்ளமைக்கப்பட்ட பண்பு ref="childComp" ஐப் பயன்படுத்தினால், பெற்றோர் கூறு குழந்தை கூறில் உள்ள தரவு பண்பு 'message' ஐக் குறியீடு this.$refs.childComp.message உடன் அணுக முடியும். (எடுத்துக்காட்டு 1 ஐப் பார்க்கவும்)

ஆனால், expose விருப்பத்தைப் பயன்படுத்தும் போது, expose விருப்பத்தில் பட்டியலிடப்பட்ட பண்புகள் மட்டுமே பெற்றோருக்குக் கிடைக்கும். (எடுத்துக்காட்டு 2 ஐப் பார்க்கவும்)

மேலும் எடுத்துக்காட்டுகள்

Example 1

குழந்தை கூறில் expose விருப்பம் பயன்படுத்தப்படவில்லை, எனவே குழந்தை கூறில் உள்ள அனைத்து பண்புகளும் பெற்றோர் கூறுக்குக் கிடைக்கும்.

ChildComp.vue:

<template>
  <div>
    <h3>ChildComp.vue</h3>
    <p>Message from parent component:</p>
    <p id="pEl">{{ message }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: ' '
    }
  },
  methods: {
    createAlert() {
      alert('This is an alert, from the child component')
    }
  }
}
</script>

<style scoped>
div {
  border: solid black 1px;
  padding: 10px;
  max-width: 350px;
  margin-top: 20px;
}
#pEl {
  background-color: lightgreen;
  font-family: 'Courier New', Courier, monospace;
}
</style>

App.vue:

<template>
  <h2>Example expose Option</h2>
  <p>The 'expose' option is not used, so all child properties are available to the parent by default, both the 'message' data property, and the 'createAlert()' method:</p>
  <button v-on:click="{ this.$refs.childComp.message += 'Hello! '; }">Write 'Hello!'</button>
  <button v-on:click="{ this.$refs.childComp.createAlert(); }">Create alert</button>
  <child-comp ref="childComp"/>
</template>

Example 2

பெற்றோர் கூறிலிருந்து குழந்தை கூறில் உள்ள 'createMessage' முறையைப் பயன்படுத்துவது வேலை செய்யாது, ஏனெனில் 'message' தரவு பண்பு மட்டுமே குழந்தை கூறின் expose விருப்பத்தில் பட்டியலிடப்பட்டுள்ளது.

ChildComp.vue:

<template>
  <div>
    <h3>ChildComp.vue</h3>
    <p>Message from parent component:</p>
    <p id="pEl">{{ message }}</p>
  </div>
</template>

<script>
export default {
  expose: ['message'],
  data() {
    return {
      message: ' '
    }
  },
  methods: {
    createMessage(msg) {
      this.message += msg + ' '
    }
  }
}
</script>

<style scoped>
div {
  border: solid black 1px;
  padding: 10px;
  max-width: 350px;
  margin-top: 20px;
}
#pEl {
  background-color: lightgreen;
  font-family: 'Courier New', Courier, monospace;
}
</style>

App.vue:

<template>
  <h2>Example expose Option</h2>
  <p>Only the 'message' data property is listed in the 'expose' option, so the 'createMessage' method from the child component is not available, and will not work:</p>
  <input type="text" v-model="inpText" placeholder="Write something">
  <button v-on:click="useMet">Use exposed method</button>
  <child-comp ref="childComp"/>
</template>

<script>
export default {
  data() {
    return {
      inpText: ''
    }
  },
  methods: {
    useMet() {
      this.$refs.childComp.createMessage(this.inpText);
    }
  },
  mounted() {
    this.$refs.childComp.message = 'Initial message!';
  }
}
</script>

Vue Expose பயிற்சி

இந்த டுடோரியலில் உள்ள விளக்கங்களைச் சரிபார்க்கக்கூடிய பயிற்சி.

❓ Vue கூறுகளில் expose விருப்பத்தின் சரியான விளக்கம் எது?

கூறை வெளிப்புற API க்கு வெளிப்படுத்த
✗ தவறு! expose விருப்பம் வெளிப்புற API உடன் தொடர்பில்லாதது
கூறின் CSS ஸ்டைல்களை கட்டுப்படுத்த
✗ தவறு! expose விருப்பம் ஸ்டைல்களுடன் தொடர்பில்லாதது
பெற்றோர் கூறிலிருந்து அனைத்து பண்புகளையும் மறைக்க
✗ தவறு! expose விருப்பம் மட்டுப்படுத்துகிறது மற்றும் மறைக்காது
டெம்ப்ளேட் refs மூலம் பெற்றோர் கூறுக்குக் கிடைக்கும் பண்புகளை மட்டுப்படுத்த
✓ சரி! expose விருப்பம் பெற்றோர் கூறுக்குக் கிடைக்கும் பண்புகளை கட்டுப்படுத்துகிறது
