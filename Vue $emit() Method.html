Vue $emit() Method

Vue.js இல் குழந்தை கூறுகளிலிருந்து பெற்றோருக்கு தரவை அனுப்ப கற்றுக்கொள்ளுங்கள்

Vue $emit() Method

Vue இல் உள்ளமைக்கப்பட்ட $emit() முறையுடன், குழந்தை கூறில் ஒரு தனிபயன் நிகழ்வை உருவாக்கலாம், இது பெற்றோர் உறுப்பில் பிடிக்கப்படும்.

புரொப்ஸ் கள் பெற்றோர் உறுப்பிலிருந்து குழந்தை கூறுக்கு தரவை அனுப்பப் பயன்படுத்தப்படுகின்றன, மேலும் $emit() எதிர் செயலுக்குப் பயன்படுத்தப்படுகிறது: குழந்தை கூறிலிருந்து பெற்றோருக்கு தகவலை அனுப்ப.

$emit() இன் நோக்கம்

நாம் அடுத்து செய்யப் போவதற்கான நோக்கம், ஒரு உணவுப் பொருளின் 'பிடித்த' நிலை App.vue பெற்றோரில் மாற்றப்படுவதாகும், அதை விட FoodItem.vue குழந்தை கூறில் அல்ல, அங்கு தற்போது மாற்றம் நடக்கிறது.

பிடித்த நிலையை FoodItem.vue க்குப் பதிலாக App.vue இல் மாற்றுவதற்கான காரணம், App.vue தான் பிடித்த நிலை முதலில் சேமிக்கப்படும் இடமாகும், எனவே அது புதுப்பிக்கப்பட வேண்டும். ஒரு பெரிய திட்டத்தில், தரவு ஒரு தரவுத்தளத்திலிருந்து வரலாம், அதை App.vue இல் இணைப்பு கொண்டுள்ளோம், மேலும் கூறில் இருந்து ஒரு மாற்றம் தரவுத்தளத்தில் மாற்றம் செய்ய வேண்டும் என விரும்புகிறோம், எனவே குழந்தை கூறிலிருந்து பெற்றோருக்குத் தகவல் தெரிவிக்க வேண்டும்.

தனிபயன் நிகழ்வை வெளியிடுதல்

கூறிலிருந்து பெற்றோருக்கு தகவலை அனுப்ப வேண்டிய அவசியம் உள்ளது, மேலும் அதைச் செய்ய நாம் உள்ளமைக்கப்பட்ட $emit() முறையைப் பயன்படுத்துகிறோம்.

FoodItem.vue கூறிற்குள் டோகிள் பொத்தான் கிளிக் செய்யப்படும் போது இயங்கும் toggleFavorite முறை எங்களிடம் ஏற்கனவே உள்ளது. இப்போது இருக்கும் வரியை அகற்றி, எங்கள் தனிபயன் நிகழ்வான 'toggle-favorite' ஐ வெளியிட ஒரு வரியைச் சேர்க்கலாம்:

// FoodItem.vue:
methods: {
  toggleFavorite() {
    this.foodIsFavorite = !this.foodIsFavorite;
    this.$emit('toggle-favorite');
  }
}

📝 குறிப்பு:

எங்கள் தனிபயன் நிகழ்வுக்கு பெயரைத் தேர்ந்தெடுக்கலாம், ஆனால் வெளியீட்டு நிகழ்வுகளுக்கு கெபாப்-வழக்கைப் பயன்படுத்துவது சாதாரணமானது.

வெளியீட்டு நிகழ்வைப் பெறுதல்

தனிபயன் வெளியீட்டு நிகழ்வு 'toggle-favorite' இப்போது FoodItem.vue கூறிலிருந்து வெளியிடப்படுகிறது, ஆனால் App.vue பெற்றோரில் நிகழ்வைக் கேட்க வேண்டும், மேலும் நிகழ்வு நடந்ததைப் பார்க்க ஒரு முறையை அழைக்க வேண்டும்.

v-on: க்குப் பதிலாக @ சுருக்கெழுத்துடன் நிகழ்வைக் கேட்கிறோம், App.vue இல் கூறு உருவாக்கப்படும் இடத்தில்:

எடுத்துக்காட்டு

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

// App.vue:
<food-item
  v-for="x in foods"
  :key="x.name"
  :food-name="x.name"
  :food-desc="x.desc"
  :is-favorite="x.favorite"
  @toggle-favorite="receiveEmit"
/>

எங்கள் தனிபயன் 'toggle-favorite' நிகழ்வு நடக்கும்போது, நிகழ்வு நடந்ததைப் பார்க்க App.vue இல் receiveEmit முறையை உருவாக்க வேண்டும்:

methods: {
  receiveEmit() {
    alert('Hello World!');
  }
}

பெற்றோரில் உணவுப் பொருளின் 'பிடித்த' நிலையை மாற்றுதல்

குழந்தை கூறிலிருந்து 'பிடித்த' பொத்தான் கிளிக் செய்யப்படும்போது App.vue க்கு அறிவிக்கும் ஒரு நிகழ்வை இப்போது வைத்திருக்கிறோம்.

ஒரு 'பிடித்த' பொத்தான் கிளிக் செய்யப்படும் போது, App.vue இல் உள்ள 'foods' வரிசையில் சரியான உணவுப் பொருளுக்கான 'பிடித்த' பண்பை மாற்ற விரும்புகிறோம். அதைச் செய்ய, FoodItem.vue இலிருந்து App.vue க்கு உணவுப் பொருள் பெயரை அனுப்புகிறோம், ஏனெனில் அது ஒவ்வொரு உணவுப் பொருளுக்கும் தனித்துவமானது:

// FoodItem.vue:
methods: {
  toggleFavorite() {
    this.$emit('toggle-favorite', this.foodName);
  }
}

'toggle-favorite' நிகழ்வு நடக்கும் போது அழைக்கப்படும் முறைக்கு ஒரு வாதமாக App.vue இல் உணவுப் பொருள் பெயரைப் பெறலாம், இது போன்றது:

எடுத்துக்காட்டு

// App.vue:
methods: {
  receiveEmit(foodId) {  
    alert( 'You clicked: ' + foodId );
  }
}

எந்த உணவுப் பொருள் கிளிக் செய்யப்பட்டது என்பது இப்போது எங்களுக்குத் தெரியும், எனவே 'foods' வரிசையில் சரியான உணவுப் பொருளுக்கான 'பிடித்த' நிலையைப் புதுப்பிக்கலாம்:

// App.vue:
methods: {
  receiveEmit(foodId) {
    const foundFood = this.foods.find(
      food => food.name === foodId
    );
    foundFood.favorite = !foundFood.favorite;
  }
}

💡 விளக்கம்:

மேலே உள்ள குறியீட்டில், 'find' வரிசை முறை 'foods' வரிசையைச் சுற்றி சென்று, பெயர் பண்பு நாம் கிளிக் செய்த உணவுப் பொருளுக்குச் சமமாக இருக்கும் ஒரு பொருளைத் தேடுகிறது, மேலும் அந்தப் பொருளை 'foundFood' ஆக வழங்குகிறது. அதன் பிறகு 'foundFood.favorite' முன்பு இருந்ததற்கு எதிர்மாறாக இருப்பதை அமைக்கலாம், இதனால் அது உண்மை மற்றும் பொய் இடையே மாறுகிறது.

படத்தைப் புதுப்பித்தல்

'foods' வரிசையிலிருந்து 'பிடித்த' நிலையுடன் உணவுப் பொருள் கூறுகள் ஏற்கனவே உருவாக்கப்பட்டுள்ளன மற்றும் App.vue இலிருந்து 'is-favorite' என ஒரு புரொப்ஸ் ஆக அனுப்பப்படுகின்றன, எனவே படத்தைப் புதுப்பிக்க உறுப்பு இருக்கும் v-show இலிருந்து FoodItem.vue இல் இந்த 'isFavorite' புரொப்ஸ் ஐக் குறிப்பிட வேண்டும்:

<img src="/img_quality.svg" v-show="isFavorite">

FoodItem.vue இல் உள்ள 'foodIsFavorite' தரவுப் பண்பையும் நீக்கலாம், ஏனெனில் அது இனி பயன்பாட்டில் இல்லை.

⚠️ கவனிக்க:

இந்த இறுதி எடுத்துக்காட்டுக் குறியீட்டில், உணவுப் பொருட்களின் பிடித்த நிலையை முன்பு போலவே மாற்றலாம், ஆனால் இப்போது பிடித்த நிலை சரியான இடத்தில், App.vue இன் உள்ளே மாற்றப்படுகிறது.

'emits' விருப்பம்

FoodItem.vue கூறிற்குள் நாம் புரொப்ஸ் களை அறிவிப்பதைப் போலவே, Vue 'emits' விருப்பத்தைப் பயன்படுத்தி கூறு என்ன வெளியிடுகிறது என்பதையும் ஆவணப்படுத்தலாம்.

புரொப்ஸ் கள் கூறில் அறிவிக்கப்பட வேண்டும், அதே நேரத்தில் வெளியீடுகள் ஆவணப்படுத்த பரிந்துரைக்கப்படுகின்றன.

FoodItem.vue கூறில் எங்கள் வெளியீட்டை எப்படி ஆவணப்படுத்தலாம்:

<script>
export default {  
  props: ['foodName','foodDesc','isFavorite'],
  emits: ['toggle-favorite'],
  methods: {
    toggleFavorite() {
      this.$emit('toggle-favorite', this.foodName);
    }
  }
};
</script>

📚 நன்மை:

வெளியீடுகள் ஆவணப்படுத்தப்படும்போது, கூறு மற்றவர்களுக்குப் பயன்படுத்த எளிதானதாகிறது.
