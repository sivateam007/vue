Composition API

Composition API உடன், நாம் Options API இலிருந்து பழக்கமான Vue நிகழ்வு கட்டமைப்பைப் பயன்படுத்துவதற்குப் பதிலாக இறக்குமதி செய்யப்பட்ட Vue செயல்பாடுகளைப் பயன்படுத்தி தர்க்கம் எழுதப்படுகிறது.

ஒரு பொத்தானுடன் சேமிப்பகத்தில் உள்ள தட்டச்சுப்பொறிகளின் எண்ணிக்கையைக் குறைக்கும் ஒரு Vue பயன்பாட்டை எழுத Composition API எவ்வாறு பயன்படுத்தப்படலாம்:

🎯 எடுத்துக்காட்டு

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

App.vue:

<template>
  <h1>Example</h1>
  <img src="/img_typewriter.jpeg" alt="Typewriter">
  <p>Typewriters left in storage: {{ typeWriters }}</p>
  <button @click="remove">Remove one</button>
  <p style="font-style: italic;">"{{ storageComment }}"</p>
</template>

<script setup>
  import { ref, computed } from 'vue'

  const typeWriters = ref(10);

  function remove(){
    if(typeWriters.value>0){
      typeWriters.value--;
    }
  }

  const storageComment = computed(
    function(){
      if(typeWriters.value > 5) {
        return "Many left"
      }
      else if(typeWriters.value > 0){
        return "Very few left"
      }
      else {
        return "No typewriters left"
      }
    }
  )
</script>

🔑 Composition API முக்கிய புள்ளிகள்:

வரி 9: setup பண்புக்கூறு Composition API பயன்படுத்துவதை எளிதாக்குகிறது. எடுத்துக்காட்டாக, setup பண்புக்கூறைப் பயன்படுத்துவதன் மூலம், மாறிகள் மற்றும் செயல்பாடுகளை நேரடியாக <template> க்குள் பயன்படுத்தலாம்.

வரி 10: ref மற்றும் computed பயன்படுத்தப்படுவதற்கு முன்பு இறக்குமதி செய்யப்பட வேண்டும். Options API இல், reactive மாறிகளை அறிவிக்க அல்லது computed பண்புகளைப் பயன்படுத்த எதையும் இறக்குமதி செய்ய தேவையில்லை.

வரி 12: ref ஆரம்ப மதிப்பாக '10' உடன் 'typewriters' பண்பை reactive ஆக அறிவிக்கப் பயன்படுகிறது.

Options API

Options API இந்த டுடோரியலின் மற்ற இடங்களில் பயன்படுத்தப்படுவது.

இந்த டுடோரியலுக்கு Options API தேர்ந்தெடுக்கப்பட்டது, ஏனெனில் அதற்கு அடையாளம் காணக்கூடிய கட்டமைப்பு உள்ளது மற்றும் தொடக்கநிலையாளர்களுடன் தொடங்குவதற்கு எளிதானதாகக் கருதப்படுகிறது.

ஒரு எடுத்துக்காட்டாக, Options API இல் உள்ள கட்டமைப்பு தரவு பண்புகள், முறைகள் மற்றும் computed பண்புகள் அனைத்தையும் தெளிவாக பிரிக்கப்பட்ட Vue நிகழ்வின் வெவ்வேறு பகுதிகளில் வைக்கிறது.

இங்கே Options API உடன் எழுதப்பட்ட மேலே உள்ள எடுத்துக்காட்டு உள்ளது:

<template>
  <h1>Example</h1>
  <img src="/img_typewriter.jpeg" alt="Typewriter">
  <p>Typewriters left in storage: {{ typeWriters }}</p>
  <button @click="remove">Remove one</button>
  <p style="font-style: italic;">"{{ storageComment }}"</p>
</template>

<script>
export default {
  data() { 
    return {
      typeWriters: 10
    };
  },
  methods: {
    remove(){
      if(this.typeWriters>0){
        this.typeWriters--;
      }
    }
  },
  computed: {
    storageComment(){
      if(this.typeWriters > 5) {
        return "Many left"
      }
      else if(this.typeWriters > 0){
        return "Very few left"
      }
      else {
        return "No typewriters left"
      }
    }
  }
}
</script>

❓ Composition API இல் reactive மாறியை உருவாக்க எந்த Vue செயல்பாடு பயன்படுத்தப்படுகிறது?

reactive()
✗ தவறு! reactive() பொருள்களுக்குப் பயன்படுத்தப்படுகிறது, ஆனால் ஒற்றை மதிப்புகளுக்கு ref() பரிந்துரைக்கப்படுகிறது
ref()
✓ சரி! ref() Composition API இல் reactive மாறிகளை உருவாக்கப் பயன்படுகிறது. இது ஒரு .value பண்புடன் ஒரு reactive குறிப்பை உருவாக்குகிறது. எடுத்துக்காட்டு: const count = ref(0); count.value++
computed()
✗ தவறு! computed() computed பண்புகளுக்குப் பயன்படுகிறது, சாதாரண reactive மாறிகளுக்கு அல்ல
data()
✗ தவறு! data() Options API இல் பயன்படுத்தப்படுகிறது, Composition API இல் அல்ல
