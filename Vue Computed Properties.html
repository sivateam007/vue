Vue Computed Properties

Vue கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் கற்றுக்கொள்ளுங்கள்

Vue Computed Properties

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் தரவு பண்புகளைப் போன்றவை, அவை மற்ற பண்புகளைச் சார்ந்திருப்பதைத் தவிர.

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் முறைகளைப் போல எழுதப்படுகின்றன, ஆனால் அவை எந்த உள்ளீட்டு வாதங்களையும் ஏற்காது.

ஒரு சார்பு மாறும்போது கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் தானாகவே புதுப்பிக்கப்படும், அதேசமயம் முறைகள் ஏதாவது நடக்கும்போது அழைக்கப்படுகின்றன, எடுத்துக்காட்டாக நிகழ்வு கையாளுதலுடன்.

வேறொன்றைச் சார்ந்துள்ள ஏதாவது வெளியிடப்படும்போது கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் பயன்படுத்தப்படுகின்றன.

Computed Properties are Dynamic

ஒரு கம்ப்யூட்டட் ப்ராபர்ட்டியின் பெரிய நன்மை என்னவென்றால், அது டைனமிக், அதாவது ஒன்று அல்லது அதற்கு மேற்பட்ட தரவு பண்புகளின் மதிப்பைப் பொறுத்து மாறுகிறது.

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் என்பது Vue உதாரணத்தில் மூன்றாவது கட்டமைப்பு விருப்பமாகும், அதை நாம் கற்றுக்கொள்வோம். நாம் ஏற்கனவே பார்த்த முதல் இரண்டு கட்டமைப்பு விருப்பங்கள் 'data' மற்றும் 'methods'.

'data' மற்றும் 'methods' போலவே கம்ப்யூட்டட் ப்ராபர்ட்டீஸ்களும் Vue உதாரணத்தில் ஒரு ஒதுக்கப்பட்ட பெயரைக் கொண்டுள்ளன: 'computed'.

📝 Syntax

கம்ப்யூட்டட் ப்ராபர்ட்டீஸின் தொடரியல்:

const app = Vue.createApp({
  data() {
    ...
  },
  computed: {
    ...
  },
  methods: {
    ...
  }
})

Computed Property 'yes' or 'no'

ஒரு ஷாப்பிங் பட்டியலில் உருப்படிகளை உருவாக்க ஒரு படிவத்தை வைத்திருக்க விரும்புகிறோம் என்று வைத்துக்கொள்வோம், மற்றும் ஒரு புதிய உருப்படி முக்கியமானதா இல்லையா எனக் குறிக்க விரும்புகிறோம். செக்பாக்ஸ் சரிபார்க்கப்படும்போது ஒரு 'true' அல்லது 'false' பின்னூட்டத்தை நாம் சேர்க்கலாம், நாம் முன்பு ஒரு உதாரணத்தில் செய்ததைப் போல:

🎯 Example

உள்ளீடு உறுப்பு டைனமிக் ஆக்கப்படுகிறது, இதனால் உரை நிலையை பிரதிபலிக்கிறது.

<input type="checkbox" v-model="chbxVal"> {{ chbxVal }}

data() {
  return {
    chbxVal: false
  }
}

இருப்பினும், ஏதாவது முக்கியமானதா என்று நீங்கள் யாரையாவது கேட்டால், அவர்கள் 'true' அல்லது 'false' க்கு பதிலாக 'yes' அல்லது 'no' என்று பதிலளிப்பார்கள். எனவே சாதாரண மொழியுடன் (மிகவும் உள்ளுணர்வு) எங்கள் படிவத்தை மிகவும் பொருத்தமாக்க, 'true' அல்லது 'false' க்கு பதிலாக செக்பாக்ஸில் 'yes' அல்லது 'no' பின்னூட்டத்தைக் கொண்டிருக்க வேண்டும்.

என்ன யூகிக்கிறீர்கள், ஒரு கம்ப்யூட்டட் ப்ராபர்ட்டி அதற்கு எங்களுக்கு உதவ ஒரு சிறந்த கருவியாகும்.

🎯 Example

ஒரு கம்ப்யூட்டட் ப்ராபர்ட்டி 'isImportant' உடன், செக்பாக்ஸ் இயக்கப்படும்போது மற்றும் அணைக்கப்படும்போது பயனருக்கு உரை பின்னூட்டத்தை இப்போது தனிப்பயனாக்கலாம்.

<input type="checkbox" v-model="chbxVal"> {{ isImportant }}

data() {
  return {
    chbxVal: false
  }
},
computed: {
  isImportant() {
    if(this.chbxVal){
      return 'yes'
    }
    else {
      return 'no'
    }
  }
}

Computed Properties vs. Methods

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் மற்றும் முறைகள் இரண்டும் செயல்பாடுகளாக எழுதப்பட்டுள்ளன, ஆனால் அவை வெவ்வேறானவை:

  • முறைகள் HTML இலிருந்து அழைக்கப்படும்போது இயக்கப்படுகின்றன, ஆனால் ஒரு சார்பு மாறும்போது கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் தானாகவே புதுப்பிக்கப்படுகின்றன.
  • கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் தரவு பண்புகளைப் பயன்படுத்தும் அதே வழியில் பயன்படுத்தப்படுகின்றன, ஆனால் அவை டைனமிக் ஆகும்.

Computed Properties

தானாகவே புதுப்பிக்கப்படுகின்றன, சார்புகள் மாறும்போது

Methods

நிகழ்வுகளால் அல்லது அழைப்புகளால் கையிருப்பு முறையில் அழைக்கப்படுகின்றன

Performance

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் முடிவுகளைக் கேஷ் செய்கின்றன, மீண்டும் கணக்கிட வேண்டியதில்லை

Usage

கம்ப்யூட்டட் ப்ராபர்ட்டீஸ் தரவு போலவே பயன்படுத்தப்படுகின்றன, () இல்லாமல்
