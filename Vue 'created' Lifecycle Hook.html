Vue 'created' Lifecycle Hook

Vue created லைஃப்சைக்கிள் ஹுக் பற்றி அறிக

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

created லைஃப்சைக்கிள் ஹுக் பயன்படுத்தி 'text' டேட்டா பிராப்பர்ட்டியை மாற்ற.

export default {
  data() {
    return {
      text: 'initial text'
    }
  },
  created() {
    this.text = 'The component is now created';
  }
}

இது எவ்வாறு செயல்படுகிறது என்பதைப் பார்க்க "Try it Yourself" பொத்தானைக் கிளிக் செய்யவும்.

வரையறை மற்றும் பயன்பாடு

created லைஃப்சைக்கிள் ஹுக் கம்போனெண்ட் துவக்கப்படும் பிறகு அழைக்கப்படுகிறது.

கம்போனெண்ட் துவக்கப்பட்டதால், டேட்டா அல்லது கம்ப்யூடட் போன்ற கம்போனெண்ட் இன்ஸ்டன்ஸ் உள்ளே உள்ள பிராப்பர்ட்டிகளை அணுகலாம், ஆனால் கம்போனெண்ட் DOM எலிமெண்டுகளை mounted ஹுக்கு வரை உருவாக்கப்படாததால் அணுக முடியாது.

Vue பயிற்சிகள்

இந்த டுடோரியலில் உள்ள பல அத்தியாயங்கள் உங்கள் அறிவு நிலையைச் சரிபார்க்கக்கூடிய பயிற்சியுடன் முடிகின்றன.

❓ created லைஃப்சைக்கிள் ஹுக் பற்றிய சரியான கூற்று எது?

created ஹுக்கில் டேட்டா பிராப்பர்ட்டிகளை அணுகலாம் மற்றும் மாற்றலாம்
✓ சரி! கம்போனெண்ட் துவக்கப்பட்ட பிறகு created ஹுக் அழைக்கப்படுவதால், டேட்டா பிராப்பர்ட்டிகளை அணுகலாம் மற்றும் மாற்றலாம்
created ஹுக்கில் DOM எலிமெண்டுகளை அணுகலாம்
✗ தவறு! created ஹுக்கில் DOM எலிமெண்டுகள் இன்னும் உருவாக்கப்படவில்லை, mounted ஹுக்குக்குப் பிறகே அவற்றை அணுக முடியும்
created ஹுக் beforeCreate ஹுக்குக்கு முன் அழைக்கப்படுகிறது
✗ தவறு! created ஹுக் beforeCreate ஹுக்குக்குப் பிறகு அழைக்கப்படுகிறது. முதலில் beforeCreate, பின்னர் created
created ஹுக்கில் this.text போன்ற பிராப்பர்ட்டிகளை அணுக முடியாது
✗ தவறு! created ஹுக்கில் கம்போனெண்ட் துவக்கப்பட்டதால் this.text போன்ற பிராப்பர்ட்டிகளை அணுகலாம் மற்றும் மாற்றலாம்
