Vue Components

Vue இல் கூறுகள் எங்கள் வலைப்பக்கத்தை சிறிய துண்டுகளாக சிதைக்க அனுமதிக்கின்றன, அவை வேலை செய்ய எளிதானவை.

வலைப்பக்கத்தின் மற்ற பகுதிகளிலிருந்து தனிமைப்படுத்தப்பட்ட ஒரு Vue கூறுடன் வேலை செய்யலாம், அதன் சொந்த உள்ளடக்கம் மற்றும் லாஜிக்குடன்.

ஒரு வலைப்பக்கம் பெரும்பாலும் பல Vue கூறுகளைக் கொண்டிருக்கும்.

What are Components?

கூறுகள் மீண்டும் பயன்படுத்தக்கூடிய மற்றும் சுய-பொறுப்புள்ள குறியீட்டின் துண்டுகள், அவை பயனர் இடைமுகத்தின் ஒரு குறிப்பிட்ட பகுதியை இணைக்கின்றன, இதனால் நாம் அளவிடக்கூடிய மற்றும் பராமரிக்க எளிதான Vue பயன்பாடுகளை உருவாக்க முடியும்.

Vue இல் கூறுகளை நாமே உருவாக்கலாம், அல்லது பின்னர் கற்றுக்கொள்ளும் உள்ளமைக்கப்பட்ட கூறுகளைப் பயன்படுத்தலாம், <Teleport> அல்லது <KeepAlive> போன்றவை. இங்கே நாம் நாமே உருவாக்கும் கூறுகளில் கவனம் செலுத்துவோம்.

Creating a Component

Vue இல் கூறுகள் மிகவும் சக்திவாய்ந்த கருவியாகும், ஏனெனில் இது எங்கள் வலைப்பக்கம் மிகவும் அளவிடக்கூடியதாக மாற அனுமதிக்கிறது மற்றும் பெரிய திட்டங்களைக் கையாள எளிதாகிறது.

ஒரு கூறை உருவாக்கி அதை எங்கள் திட்டத்தில் சேர்ப்போம்.

  1. src கோப்புறையின் உள்ளே components என்ற புதிய கோப்புறையை உருவாக்கவும்.
  2. components கோப்புறையின் உள்ளே, FoodItem.vue என்ற புதிய கோப்பை உருவாக்கவும். பாஸ்கல் கேஸ் பெயரிடல் மரபுடன் கூறுகளைப் பெயரிடுவது பொதுவானது, இடைவெளிகள் இல்லாமல் மற்றும் அனைத்து புதிய சொற்களும் ஒரு பெரிய எழுத்தில் தொடங்கும், முதல் சொல்லும்.
  3. FoodItem.vue கோப்பு இப்படி இருக்கும் என்பதை உறுதிப்படுத்தவும்:

📝 FoodItem.vue component:

FoodItem.vue கூறுக்குள் உள்ள குறியீடு:

<template>
  <div>
    <h2>{{ name }}</h2>
    <p>{{ message }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      name: 'Apples',
      message: 'I like apples'
    }
  }
};
</script>

<style></style>

மேலே உள்ள உதாரணத்தில் நீங்கள் பார்க்க முடியும், கூறுகளும் <template>, <script> மற்றும் <style> குறிச்சொற்களைக் கொண்டிருக்கின்றன, எங்கள் முதன்மை App.vue கோப்பைப் போலவே.

Adding The Component

மேலே உள்ள உதாரணத்தில் <script> குறிச்சொல் export default உடன் தொடங்குகிறது என்பதைக் கவனியுங்கள். இதன் பொருள் தரவு பண்புகளைக் கொண்ட பொருள் மற்றொரு கோப்பில் பெறப்படலாம் அல்லது இறக்குமதி செய்யப்படலாம். இதை main.js கோப்புடன் இறக்குமதி செய்வதன் மூலம் FoodItem.vue கூறை எங்கள் இருக்கும் திட்டத்தில் செயல்படுத்த இதைப் பயன்படுத்துவோம்.

முதலில், உங்கள் அசல் main.js கோப்பில் கடைசி வரியை இரண்டு வரிகளாக மீண்டும் எழுதவும்:

📝 main.js

புதிய main.js கோப்பு:

import { createApp } from 'vue'

import App from './App.vue'

const app = createApp(App)
app.mount('#app')

இப்போது, உங்கள் main.js கோப்பில் வரி 4 மற்றும் 7 ஐச் செருவதன் மூலம் FoodItem.vue கூறைச் சேர்க்கவும்:

import { createApp } from 'vue'

import App from './App.vue'
import FoodItem from './components/FoodItem.vue'

const app = createApp(App)
app.component('food-item', FoodItem)
app.mount('#app')

வரி 7 இல், கூறு சேர்க்கப்பட்டுள்ளது, இதனால் அதை எங்கள் App.vue கோப்பில் <template> குறிச்சொல்லின் உள்ளே ஒரு தனிப்பயன் குறிச்சொல்லாக <food-item/> பயன்படுத்தலாம், இப்படி:

<template>
  <h1>Food</h1>
  <food-item/>
  <food-item/>
  <food-item/>
</template>

<script></script>

<style></style>

மேலும், App.vue கோப்பில் <style> குறிச்சொல்லின் உள்ளே சில ஸ்டைலிங்கைச் சேர்ப்போம். டெவலப்ப்மென்ட் சேவையகம் இயங்குவதை உறுதிப்படுத்தவும், மற்றும் முடிவைப் பார்க்கவும்.

🎯 Example

முழுமையான App.vue எடுத்துக்காட்டு:

<template>
  <h1>Food</h1>
  <food-item/>
  <food-item/>
  <food-item/>
</template>

<script></script>

<style>
  #app > div {
    border: dashed black 1px;
    display: inline-block;
    margin: 10px;
    padding: 10px;
    background-color: lightgreen;
  }
</style>

💻 Development mode:

உங்கள் Vue திட்டங்களுடன் பணிபுரியும் போது, டெர்மினலில் பின்வரும் குறியீட்டு வரியை இயக்குவதன் மூலம் எப்போதும் உங்கள் திட்டத்தை டெவலப்ப்மென்ட் பயன்முறையில் வைத்திருப்பது பயனுள்ளதாக இருக்கும்:

npm run dev

Individual Components

Vue இல் கூறுகளுடன் பணிபுரியும் போது ஒரு மிகவும் பயனுள்ள மற்றும் சக்திவாய்ந்த பண்பு என்னவென்றால், நாம் அவற்றை தனிப்பட்ட முறையில் செயல்பட வைக்கலாம், வெற்று ஜாவாஸ்கிரிப்ட்டுடன் நாம் செய்ய வேண்டியதைப் போல தனிப்பட்ட ID களுடன் கூறுகளைக் குறிக்க வேண்டியதில்லை. Vue ஒவ்வொரு கூறையும் தனித்தனியாக கையாள தானாகவே கவனித்துக்கொள்கிறது.

<div> கூறுகளை கிளிக் செய்யும்போது எண்ணுவோம்.

எங்கள் முதன்மை பயன்பாட்டு கோப்பான App.vue க்கு சேர்க்கப்பட்ட ஒரே விஷயம் CSS இல் கர்சர் ஒரு கை சுட்டிக்காட்டுவது போல் தோற்றமளிக்கும், ஹோவர் செய்யும் போது கிளிக் செயல்பாடு ஏதோ இருக்கிறது என்பதை குறிப்பிடுவதாகும்.

#app > div:hover {
  cursor: pointer;
}

எங்கள் கூறு கோப்பு FoodItem.vue இல் நாம் ஒரு தரவு பண்பு count, <div> உறுப்புக்கு ஒரு கிளிக் கேட்பவர், கிளிக் நடக்கும்போது இயக்க ஒரு முறை, மற்றும் எண்ணிக்கையைக் காட்ட உரை இடைசெருகல் {{}} ஆகியவற்றைச் சேர்க்க வேண்டும்.

🎯 Example

முழுமையான FoodItem.vue எடுத்துக்காட்டு:

<template>
  <div v-on:click="countClicks">
    <h2>{{ name }}</h2>  
    <p>{{ message }}</p>
    <p id="red">You have clicked me {{ clicks }} times.</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      name: 'Apples',
      message: 'I like apples',
      clicks: 0
    }
  },
  methods: {
    countClicks() {
      this.clicks++;
    }
  }
}
</script>

<style>
  #red {
    font-weight: bold ;
    color: rgb(144, 12, 12);
  }
</style>

Vue க்கு ஒவ்வொரு <div> உறுப்புக்கும் தனித்தனியாக எண்ணிக்கையைக் கையாள தனிப்பட்ட ID களை வரையறுக்க வேண்டியதில்லை அல்லது கூடுதல் வேலை எதுவும் செய்ய வேண்டியதில்லை, Vue இது தானாகவே செய்கிறது.

ஆனால் வெவ்வேறு கவுண்டர் மதிப்புகளைத் தவிர, <div> கூறுகளின் உள்ளடக்கம் இன்னும் ஒன்றே. அடுத்த பக்கத்தில் கூறுகள் பற்றி மேலும் அறிவோம், இதனால் கூறுகளை அதிக அர்த்தமுள்ள வழியில் பயன்படுத்த முடியும். உதாரணமாக, ஒவ்வொரு <div> உறுப்பிலும் வெவ்வேறு வகையான உணவைக் காட்டுவது அதிக அர்த்தமுள்ளதாக இருக்கும்.

Vue Exercises

main.js இல் உள்ள கீழே உள்ள வரி எங்கள் Vue திட்டத்தில் ஒரு கூறைச் சேர்க்கிறது:

app.component('fish-type', FishType)

இந்த கூறை App.vue இல் எவ்வாறு சேர்க்கலாம்?

❓ Fill in the missing component tag:

<template>
  <h1>Fish</h1>
  ______
</template>
<fish-type/>
✓ சரி! 'fish-type' என்பது app.component() இல் பதிவுசெய்யப்பட்ட கூறு பெயர், எனவே <fish-type/> குறிச்சொல் App.vue இல் பயன்படுத்தப்படும்
<FishType/>
✗ தவறு! கூறு 'fish-type' என பதிவுசெய்யப்பட்டுள்ளது, 'FishType' அல்ல. Vue கூறு குறிச்சொற்கள் பொதுவாக கேபப்-கேஸில் இருக்கும்
<fishType/>
✗ தவறு! கூறு பெயர் 'fish-type' என பதிவுசெய்யப்பட்டுள்ளது, 'fishType' அல்ல. கூறு பெயர் சரியாக பொருந்த வேண்டும்
<Fish-type/>
✗ தவறு! கூறு குறிச்சொற்கள் பொதுவாக கேபப்-கேஸில் இருக்கும், கேமல் கேஸில் அல்ல. 'fish-type' சரியான குறிச்சொல்
