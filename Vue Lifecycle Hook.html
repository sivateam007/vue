Vue லைப்சைக்கிள் ஹுக்குகளைக் கற்றுக்கொள்ளுங்கள்

Vue லைப்சைக்கிள் ஹுக்குகள்

Vue இல் உள்ள லைப்சைக்கிள் ஹுக்குகள் என்பது கம்போனென்ட்டின் வாழ்க்கைச் சுழற்சியில் உள்ள சில கட்டங்களாகும், அங்கு நாம் விஷயங்களைச் செய்ய குறியீட்டைச் சேர்க்கலாம்.

லைப்சைக்கிள் ஹுக்குகள்

ஒவ்வொரு முறையும் ஒரு கம்போனென்ட் அதன் வாழ்க்கைச் சுழற்சியில் ஒரு புதிய கட்டத்தை அடையும் போது, ஒரு குறிப்பிட்ட செயல்பாடு இயங்குகிறது, மேலும் நாம் அந்த செயல்பாட்டில் குறியீட்டைச் சேர்க்கலாம். இத்தகைய செயல்பாடுகள் லைப்சைக்கிள் ஹுக்குகள் என்று அழைக்கப்படுகின்றன, ஏனெனில் நாம் அந்த கட்டத்தில் எங்கள் குறியீட்டை "ஹுக்" செய்யலாம்.

ஒரு கம்போனென்ட் கொண்டிருக்கும் அனைத்து லைப்சைக்கிள் ஹுக்குகள்:

'beforeCreate' ஹுக்

beforeCreate லைப்சைக்கிள் ஹுக் கம்போனென்ட் துவக்கப்படுவதற்கு முன் நடக்கும், எனவே இது Vue கம்போனென்ட்டின் தரவு, கம்ப்யூடட் ப்ராபர்ட்டிகள், மெத்தட்கள் மற்றும் இவெண்ட் லிஸ்டனர்களை அமைக்கும் முன்பு உள்ளது.

beforeCreate ஹுக் உதாரணமாக ஒரு குளோபல் இவெண்ட் லிஸ்டனரை அமைக்க பயன்படுத்தப்படலாம், ஆனால் beforeCreate லைப்சைக்கிள் ஹுக்கிலிருந்து கம்போனென்ட்டுக்கு சொந்தமான உறுப்புகளை அணுக முயற்சிப்பதைத் தவிர்க்க வேண்டும், ஏனெனில் அவை இந்த கட்டத்தில் இன்னும் உருவாக்கப்படவில்லை.

மேலும், beforeCreate லைப்சைக்கிள் ஹுக்கிலிருந்து DOM உறுப்புகளை அணுக முயற்சிப்பது அர்த்தமற்றது, ஏனெனில் அவை கம்போனென்ட் மவுண்ட் செய்யப்பட்ட பிறகுதான் உருவாக்கப்படுகின்றன.

எடுத்துக்காட்டு

உங்கள் சொந்த Vue சேவையகத்தைப் பெறுங்கள்

CompOne.vue:
<template>
    <h2>Component</h2>
    <p>This is the component</p>
    <p id="pResult">{{ text }}</p>
</template>

<script>
export default {
    data() {
        return {
            text: '...'
        }
    },
  beforeCreate() {
        this.text = 'initial text'; // This line has no effect
    console.log("beforeCreate: The component is not created yet.");
  }
}
</script>
App.vue:
<template>
  <h1>The 'beforeCreate' Lifecycle Hook</h1>
  <p>We can see the console.log() message from 'beforeCreate' lifecycle hook, but there is no effect from the text change we try to do to the Vue data property, because the Vue data property is not created yet.</p>
  <button @click="this.activeComp = !this.activeComp">Add/Remove Component</button>
  <div>
    <comp-one v-if="activeComp"></comp-one>
  </div>
</template>

<script>
export default {
  data() {
    return {
      activeComp: false
    }
  }
}
</script>

'created' ஹுக்

created லைப்சைக்கிள் ஹுக் கம்போனென்ட் துவக்கப்பட்ட பிறகு நடக்கும், எனவே Vue ஏற்கனவே கம்போனென்ட்டின் தரவு, கம்ப்யூடட் ப்ராபர்ட்டிகள், மெத்தட்கள் மற்றும் இவெண்ட் லிஸ்டனர்களை அமைத்துள்ளது.

கம்போனென்ட் மவுண்ட் செய்யப்படும் வரை DOM உறுப்புகள் அணுக முடியாது என்பதால், created லைப்சைக்கிள் ஹுக்கிலிருந்து DOM உறுப்புகளை அணுக முயற்சிப்பதைத் தவிர்க்க வேண்டும்.

created லைப்சைக்கிள் ஹுக் HTTP கோரிக்கைகளுடன் தரவைப் பெற அல்லது ஆரம்ப தரவு மதிப்புகளை அமைக்க பயன்படுத்தப்படலாம். கீழே உள்ள எடுத்துக்காட்டில் உள்ளது போல, தரவு ப்ராபர்ட்டி 'text' ஒரு ஆரம்ப மதிப்பை வழங்குகிறது:

'beforeMount' ஹுக்

beforeMount லைப்சைக்கிள் ஹுக் கம்போனென்ட் மவுண்ட் செய்யப்படுவதற்கு சற்று முன்பு நடக்கும், எனவே கம்போனென்ட் DOM இல் சேர்க்கப்படுவதற்கு சற்று முன்பு.

கம்போனென்ட் மவுண்ட் செய்யப்படும் வரை DOM உறுப்புகள் அணுக முடியாது என்பதால், beforeMount லைப்சைக்கிள் ஹுக்கிலிருந்து DOM உறுப்புகளை அணுக முயற்சிப்பதைத் தவிர்க்க வேண்டும்.

கீழே உள்ள எடுத்துக்காட்டு கம்போனென்ட்டில் DOM உறுப்புகளை இன்னும் அணுக முடியாது என்பதைக் காட்டுகிறது, CompOne.vue இல் வரி 11 வேலை செய்யாது, மற்றும் பிரௌசர் கன்சோலில் ஒரு பிழையை உருவாக்குகிறது:

'mounted' ஹுக்

ஒரு கம்போனென்ட் DOM மரத்தில் சேர்க்கப்பட்ட உடனேயே, mounted() செயல்பாடு அழைக்கப்படுகிறது, மேலும் நாம் அந்த கட்டத்தில் எங்கள் குறியீட்டைச் சேர்க்கலாம்.

கம்போனென்ட்டுக்குச் சொந்தமான DOM உறுப்புகள் தொடர்பான விஷயங்களைச் செய்ய இது முதல் வாய்ப்பாகும், கீழே உள்ள இரண்டாவது எடுத்துக்காட்டில் நாம் செய்வது போல ref பண்பு மற்றும் $refs பொருளைப் பயன்படுத்துதல்.

எடுத்துக்காட்டு 1

CompOne.vue:
<template>
  <h2>Component</h2>
  <p>Right after this component is added to the DOM, the mounted() function is called and we can add code to that mounted() function. In this example, an alert popup box appears after this component is mounted.</p>
  <p><strong>Note:</strong> The reason that the alert is visible before the component is visible is because the alert is called before the browser gets to render the component to the screen.</p>
</template>

<script>
export default {
  mounted() {
    alert("The component is mounted!");
  }
}
</script>

மவுண்ட் செய்யப்பட்ட கட்டம் கம்போனென்ட் DOM இல் சேர்க்கப்பட்ட பிறகு நடக்கிறது, ஆனால் மேலே உள்ள எடுத்துக்காட்டில் alert() கம்போனென்ட் தெரியும் முன்பே தெரியும். இதற்கான காரணம் முதலில் கம்போனென்ட் DOM இல் சேர்க்கப்படுகிறது, ஆனால் பிரௌசர் கம்போனென்ட்டை திரையில் ரெண்டர் செய்யும் முன், மவுண்ட் செய்யப்பட்ட கட்டம் நடக்கிறது மற்றும் alert() தெரியும் மற்றும் கம்போனென்ட்டை ரெண்டர் செய்வதில் பிரௌசரை நிறுத்துகிறது.

<template>

'beforeUpdate' ஹுக்

beforeUpdate லைப்சைக்கிள் ஹுக் எங்கள் கம்போனென்ட்டின் தரவில் ஏதேனும் மாற்றம் இருக்கும்போதெல்லாம் அழைக்கப்படுகிறது, ஆனால் புதுப்பிப்பு திரையில் ரெண்டர் செய்யப்படுவதற்கு முன்பு. beforeUpdate லைப்சைக்கிள் ஹுக் updated லைப்சைக்கிள் ஹுக்கிற்கு சற்று முன்பு நடக்கிறது.

beforeUpdate ஹுக்கைப் பற்றிய சில சிறப்பு விஷயம் என்னவென்றால், புதிய புதுப்பிப்பைத் தூண்டாமல் நாம் பயன்பாட்டில் மாற்றங்களைச் செய்யலாம், எனவே இல்லையெனில் முடிவில்லா சுழற்சியைத் தவிர்க்கலாம். updated லைப்சைக்கிள் ஹுக்கில் பயன்பாட்டில் மாற்றங்களைச் செய்யக்கூடாது என்பதற்கு இதுவே காரணம், ஏனெனில் அந்த ஹுக்குடன், ஒரு முடிவில்லா சுழற்சி உருவாக்கப்படும். கீழே இருந்து மூன்றாவது எடுத்துக்காட்டைப் பாருங்கள், சிவப்பு நிறத்தில்.

எடுத்துக்காட்டு

beforeUpdate() செயல்பாடு beforeUpdate() செயல்பாடு இயங்கியுள்ளது என்பதைக் குறிக்க ஆவணத்தில் ஒரு <li> டேக்கைச் சேர்க்கிறது.

CompOne.vue:
<template>
  <h2>Component</h2>
  <p>This is the component</p>
</template>
App.vue:
<template>
  <h1>The 'beforeUpdate' Lifecycle Hook</h1>
  <p>Whenever there is a change in our page, the application is 'updated' and the 'beforeUpdate' hook happens just before that.</p>
  <p>It is safe to modify our page in the 'beforeUpdate' hook like we do here, but if we modify our page in the 'updated' hook, we will generate an infinite loop.</p>
  <button @click="this.activeComp = !this.activeComp">Add/Remove Component</button>
  <div>
    <comp-one v-if="activeComp"></comp-one>
  </div>
  <ol ref="divLog"></ol>
</template>

<script>
export default {
  data() {
    return {
      activeComp: true
    }
  },
  beforeUpdate() {
    this.$refs.divLog.innerHTML += "<li>beforeUpdate: This happened just before the 'updated' hook.</li>";
  }
}
</script>

<style>
#app > div {

'renderTracked' மற்றும் 'renderTriggered' லைப்சைக்கிள் ஹுக்குகள்

renderTracked ஹுக் ஒரு ரெண்டர் செயல்பாடு ஒரு ரியாக்டிவ் கம்போனென்டைக் கண்காணிக்க அல்லது கண்காணிக்க அமைக்கப்படும் போது இயங்கும். renderTracked ஹுக் வழக்கமாக ஒரு ரியாக்டிவ் கம்போனென்ட் துவக்கப்படும் போது இயங்கும்.

renderTriggered ஹுக் அத்தகைய கண்காணிக்கப்பட்ட ரியாக்டிவ் கம்போனென்ட் மாறும்போது இயங்கும், எனவே ஒரு புதிய ரெண்டரைத் தூண்டுகிறது, இதனால் திரை சமீபத்திய மாற்றங்களுடன் புதுப்பிக்கப்படுகிறது.

ரியாக்டிவ் கம்போனென்ட் என்பது மாறக்கூடிய ஒரு கம்போனென்ட் ஆகும்.

ரெண்டர் செயல்பாடு என்பது Vue ஆல் தொகுக்கப்பட்ட ஒரு செயல்பாடாகும், இது ரியாக்டிவ் கம்போனென்ட்களைக் கண்காணிக்கிறது. ஒரு ரியாக்டிவ் கம்போனென்ட் மாறும்போது, ரெண்டர் செயல்பாடு தூண்டப்படுகிறது மற்றும் பயன்பாட்டை திரையில் மீண்டும் ரெண்டர் செய்கிறது.

renderTracked மற்றும் renderTriggered ஹுக்கள் டீபகிஙிற்காக மட்டுமே பயன்படுத்தப்படும், மேலும் அவை டெவலப்மென்ட் மோட்டில் மட்டுமே கிடைக்கின்றன.

'activated' மற்றும் 'deactivated' லைப்சைக்கிள் ஹுக்குகள்

இந்தப் பக்கத்தில் மேலே நாம் பார்க்க முடியும் என, ஒரு கம்போனென்ட் DOM இலிருந்து அகற்றப்படும் அல்லது சேர்க்கப்படும் போது mounted மற்றும் unmounted லைப்சைக்கிள் ஹுக்குகள் உள்ளன.

activated மற்றும் deactivated லைப்சைக்கிள் ஹுக்கள் ஒரு கேச் செய்யப்பட்ட டைனமிக் கம்போனென்ட் சேர்க்கப்படும் அல்லது அகற்றப்படும் போது, ஆனால் DOM இலிருந்து அல்ல. கீழே உள்ள எடுத்துக்காட்டில் <KeepAlive> டேக் டைனமிக் கம்போனென்டை கேச் செய்ய பயன்படுத்தப்படுகிறது.

எடுத்துக்காட்டு

CompOne.vue:
<template>
  <h2>Component</h2>
  <p>Below is a log with every time the 'activated', 'deactivated', 'mounted' or 'unmounted' hooks run.</p>
  <ol ref="olEl"></ol>
  <p>You can also see when these hooks run in the console.</p>
</template>
  
<script>
export default {
  mounted() {
    this.logHook("mounted");
  },
  unmounted() {
    this.logHook("unmounted");
  },
  activated() {
    this.logHook("activated");
  },
  deactivated() {
    this.logHook("deactivated");
  },
  methods: {
    logHook(hookName) {
      console.log(hookName);
      const liEl = document.createElement("li");
      liEl.innerHTML = hookName;
      this.$refs.olEl.appendChild(liEl);
    }
  }
}
</script>

<style>
  li {
    background-color: lightcoral;
    width: 5em;
  }
</style>
App.vue:
<template>
  <h1>The 'activated' and 'deactivated' Lifecycle Hooks</h1>
  <p>In this example for the 'activated' and 'deactivated' hooks we also see when and if the 'mounted' and 'unmounted' hooks are run.</p>
  <button @click="this.activeComp = !this.activeComp">Include component</button>
  <div>
    <KeepAlive>
      <comp-one v-if="activeComp"></comp-one>
    </KeepAlive>
  </div>
</template>

<script>
export default {
  data() {
    return {
      activeComp: false
    }
  }
}
</script>

'serverPrefetch' லைப்சைக்கிள் ஹுக்

'serverPrefetch' ஹுக் சர்வர்-சைட் ரெண்டரிங் (SSR) போது மட்டுமே அழைக்கப்படுகிறது.

'serverPrefetch' ஹுக்கிற்கான விளக்கம் மற்றும் எடுத்துக்காட்டை உருவாக்குவதற்கு ஒப்பீட்டளவில் நீண்ட அறிமுகம் மற்றும் அமைப்பு தேவைப்படும், மேலும் அது இந்த டுடோரியலின் நோக்கத்திற்கு அப்பாற்பட்டது.
